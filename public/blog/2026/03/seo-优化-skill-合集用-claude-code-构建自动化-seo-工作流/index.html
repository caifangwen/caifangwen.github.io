# 关键词研究 Skill

## 工作流

### 步骤 1：确认研究范围
- 目标行业 / 产品类别
- 核心种子词（1-5 个）
- 地区和语言（默认：中文简体，中国大陆）
- 目标页面类型（首页 / 产品页 / 博客 / 落地页）

### 步骤 2：关键词扩展矩阵

按以下四象限扩展：

| 类型 | 特征 | 示例 |
|------|------|------|
| 品牌词 | 包含品牌名 | "品牌名 + 功能" |
| 产品词 | 直接描述产品 | "XX 软件购买" |
| 信息词 | 用户学习阶段 | "XX 是什么" / "如何使用XX" |
| 竞品词 | 竞争对手相关 | "XX 替代品" / "XX vs YY" |

### 步骤 3：搜索意图分类（TOFU/MOFU/BOFU）

- **TOFU（漏斗顶部）**：泛信息词，触达潜在用户
- **MOFU（漏斗中部）**：方案对比词，影响考量决策
- **BOFU（漏斗底部）**：购买意图词，驱动转化

### 步骤 4：竞争难度评估

对每个关键词评估：
- 搜索意图明确度（1-5 分）
- 内容产出可行性（是否有独特价值可提供）
- 建议优先级（高/中/低）

### 步骤 5：输出关键词报告

生成结构化 Markdown 表格，包含：
- 关键词 | 意图类型 | 漏斗阶段 | 优先级 | 建议内容形式

## 输出规范

- 输出为 Markdown 表格 + 简要策略说明
- 关键词数量：种子词 ×10 扩展（每类至少 3 个）
- 必须包含：高优先级关键词的内容建议

# 内容 SEO 优化 Skill

## 前置条件

获取以下信息后再开始优化：
- [ ] 目标关键词（主词 + 2-3 个相关词）
- [ ] 目标读者画像
- [ ] 当前文章内容（粘贴或上传）
- [ ] 竞争对手 URL（可选，用于差异化分析）

## 核心工作流

### 步骤 1：SEO 健康诊断

检查以下 12 项指标并打分（✅/⚠️/❌）：

**标题优化层**
- [ ] H1 标签包含主关键词
- [ ] H1 长度 20-60 字符
- [ ] H2/H3 覆盖相关词和问题词

**Meta 信息层**
- [ ] Title Tag：含主词，50-60 字符
- [ ] Meta Description：含行动召唤，150-160 字符

**内容质量层**
- [ ] 关键词自然分布（首段、小标题、结尾）
- [ ] 内容深度（字数是否达到竞争水平）
- [ ] 原创性和独特价值点
- [ ] FAQ 段落（覆盖 People Also Ask）

**内部链接层**
- [ ] 是否有 3+ 个内链锚文本机会
- [ ] 图片 Alt 标签优化
- [ ] 结构化数据建议

### 步骤 2：输出优化版本

按优先级输出：
1. **必改**：标题、Meta、H2 结构
2. **建议**：段落改写、FAQ 添加
3. **可选**：内链建议、Schema 标记

## 输出规范

每项优化提供：原文 → 优化版本 + 改动理由

# 技术 SEO 审计 Skill

## 审计范围选择

根据用户输入决定审计深度：

| 场景 | 审计类型 | 重点 |
|------|----------|------|
| 新站上线前 | 基础检查 | robots/sitemap/HTTPS/索引 |
| 流量下降排查 | 深度诊断 | 爬取/重定向/规范化/惩罚 |
| 全站健康检查 | 完整审计 | 所有技术指标 |
| 性能优化 | 速度专项 | CWV/图片/JS/CSS |

## 核心工作流

### 步骤 1：可索引性检查
- 抓取 robots.txt，检查是否误封重要目录
- 验证 XML Sitemap 格式和更新时效
- 检查 noindex/nofollow 使用是否合理
- 确认 canonical 标签指向正确

### 步骤 2：技术健康指标
- HTTPS 部署（混合内容问题）
- 移动端适配（viewport/响应式）
- 页面加载速度指标（CWV：LCP/FID/CLS）
- 重定向链（超过 2 跳需处理）
- 404 错误页面处理

### 步骤 3：结构化数据验证
- 检查 Schema.org 标记类型
- 验证 JSON-LD 格式正确性
- 推荐适合该站类型的 Schema

### 步骤 4：输出审计报告

按严重程度分类：
- 🔴 **紧急修复**（影响索引或排名）
- 🟡 **优化建议**（影响用户体验）
- 🟢 **最佳实践**（锦上添花）

每项问题提供：问题描述 → 影响范围 → 修复方案 → 优先级

## 输出规范

生成结构化审计报告，包含：执行摘要、问题清单、修复路线图

# 外链机会挖掘 Skill

## 核心工作流

### 步骤 1：竞品外链格局分析

通过搜索确定：
- 行业内高 DA（域名权重）的参考站点
- 竞品普遍拥有但我方缺失的链接类型
- 本行业常见的外链获取渠道

### 步骤 2：外链机会分类

| 机会类型 | 获取难度 | 质量 | 适用场景 |
|----------|----------|------|----------|
| 资源页链接 | 中 | 高 | 工具类/指南类内容 |
| 嘉宾博客 | 高 | 高 | 有专家背书的品牌 |
| 品牌提及追踪 | 低 | 中 | 已有媒体曝光的品牌 |
| 损坏链接替换 | 中 | 高 | 有高质量替代内容 |
| 行业目录收录 | 低 | 中低 | 新站快速建立基础 |
| 数据/研究引用 | 高 | 极高 | 有原创数据的内容 |

### 步骤 3：推广材料准备建议

针对目标外链类型，建议准备：
- 外联邮件模板（个性化变量清单）
- 价值主张清单（为什么对方应该链接你）
- 目标站点筛选标准（DA阈值/行业相关性/流量要求）

### 步骤 4：输出外链策略报告

包含：机会列表、优先级排序、行动计划

# SEO 报告生成 Skill

1. **执行摘要**：3 句话总结本期核心成果

总结

这套 SEO Skill 合集覆盖了 SEO 工作的完整链路：

Skill 核心价值 适用频率
关键词研究 内容策略基石，节省人工扩词时间 80% 月度/季度
内容优化 存量内容提效，每篇节省 2-4 小时 周度
技术审计 消除技术债，避免排名损失 月度/按需
外链挖掘 系统化外链策略，替代零散操作 月度
报告生成 汇报效率提升 70%，数据洞察更深 月度/季度

Skill 体系的核心价值在于将 SEO 专家的隐性经验显式化：不管是新手还是老手，每次执行都能稳定输出专业水准的 SEO 分析结果——这正是 AI 辅助工作流从"偶尔惊艳"走向"稳定可靠"的关键路径。
