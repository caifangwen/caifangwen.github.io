## Hugo 支持的日期格式

Hugo 支持以下几种日期格式:

### 1. ISO 8601 完整格式 (推荐)
```yaml
date: 2026-01-27T12:00:00+08:00
```
- 包含日期、时间和时区
- 最完整和明确的格式
- 适合需要精确时间的场景

### 2. 日期加时间
```yaml
date: 2026-01-27 12:00:00
```
- 没有时区信息
- Hugo 会使用默认时区

### 3. 仅日期格式
```yaml
date: 2026-01-27
```
- 最简洁的格式
- Hugo 会自动添加默认时间 (00:00:00)
- 适合只需要日期不需要具体时间的场景

### 4. RFC 3339 格式
```yaml
date: 2026-01-27T12:00:00Z
```
- Z 表示 UTC 时间

## ❌ 不支持的日期格式

以下格式 Hugo **不支持**,会导致错误:

```yaml
date: 1-27-2026        # ❌ 错误: 月-日-年格式
date: 27/01/2026       # ❌ 错误: 使用斜杠分隔
date: Jan 27, 2026     # ❌ 错误: 英文月份名称
date: 2026年1月27日    # ❌ 错误: 中文格式
```

## 建议的最佳实践

### 对于中文博客
推荐使用带时区的完整格式:
```yaml
date: 2026-01-27T12:00:00+08:00
```
- `+08:00` 表示中国标准时间(东八区)
- 这样在部署到国外服务器时也能保持正确的时间

### 对于简单文章
如果不需要精确时间,可以使用简化格式:
```yaml
date: 2026-01-27
```

## 预防措施

为避免将来出现类似问题,建议:

1. **创建文章模板** (在 `archetypes/default.md` 中):
   ```yaml
   ---
   title: "{{ replace .Name "-" " " | title }}"
   date: {{ .Date }}
   draft: true
   description: ""
   keywords: []
   author: "Frida"
   tags: []
   categories: []
   ---
   ```

2. **使用 Hugo 命令创建新文章:**
   ```bash
   hugo new posts/my-new-post.md
   ```
   这样会自动生成正确格式的日期

3. **编辑器配置:**
   如果使用 VS Code,可以安装 Hugo 插件来验证 front matter 格式
