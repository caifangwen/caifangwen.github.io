## 概览：为什么 SEM 需要 Skill 体系？

Google SEM（搜索引擎营销）是高度专业化的领域，涉及关键词策略、广告文案、出价逻辑、质量得分、转化追踪等多个专业模块。每个模块都有固定的最佳实践和输出规范——这正是 Skill 体系发挥价值的最佳场景。

本合集基于[《如何构建 AI Skill 体系》](/posts/how-to-build-skill-system)中的设计哲学，构建一套**项目级** Google SEM Skill，可直接部署至 Claude Code 工作流。

# Google SEM 关键词研究 Skill

## 概述
系统化完成关键词研究四步流程：种子词扩展 → 竞争评估 → 分组策略 → 匹配类型规划。
最终输出结构化关键词表格（.xlsx 或 Markdown 表格）。

## 前置条件
执行前确认以下信息（若未提供则主动询问）：
- [ ] 产品/服务核心描述（1-2 句话）
- [ ] 目标受众画像
- [ ] 月度预算范围（影响词量规模建议）
- [ ] 是否有现有关键词列表可作为输入

## 核心工作流

### 步骤 1：种子词扩展
从用户提供的产品描述出发，生成种子词矩阵：

| 维度 | 示例（以"CRM软件"为例） |
|------|----------------------|
| 品牌词 | 公司名、产品名 |
| 通用词 | crm软件、客户管理系统 |
| 功能词 | 销售漏斗管理、客户跟进工具 |
| 场景词 | 销售团队管理软件、B2B客户管理 |
| 竞品词 | [竞品名] 替代品（需确认预算允许） |
| 痛点词 | 客户流失怎么办、销售效率低 |
| 长尾词 | 小企业crm软件哪个好用 |

**输出**：初始种子词列表（≥50 个）

### 步骤 2：竞争度与商业价值评估
对每个关键词标注：

| 字段 | 说明 |
|------|------|
| 搜索意图 | 信息型 / 导航型 / 交易型 / 商业调研型 |
| 竞争程度 | 低 / 中 / 高（基于词的商业价值判断） |
| 购买阶段 | 认知期 / 考虑期 / 决策期 |
| 优先级 | P1（核心必投）/ P2（扩量）/ P3（长尾测试） |

### 步骤 3：关键词分组（Ad Group 规划）
遵循 SKAG（Single Keyword Ad Group）或 STAG 策略，
详见 `references/skag-vs-stag.md`（campaign-structure Skill 中）。

分组原则：
- 同一组内词的搜索意图高度一致
- 同一组词可共享一套广告文案
- 组内词量：SKAG = 1-3 词；STAG = 5-20 词

### 步骤 4：匹配类型规划
读取 `references/match-types.md`，为每组词分配匹配类型组合：

| 阶段 | 推荐组合 | 逻辑 |
|------|----------|------|
| 新账户冷启动 | 词组匹配 + 精确匹配 | 控制流量质量，积累数据 |
| 优化期 | 精确匹配为主 + 广泛（智能出价） | 智能扩量 |
| 扩量期 | 广泛匹配 + 智能出价 | 让算法探索新词 |

### 步骤 5：否定关键词清单
读取 `references/negative-keywords.md`，生成：
- 账户级否定词（通用品牌保护词、竞品否定等）
- 广告系列级否定词（跨系列流量隔离）
- 行业通用否定词（如"免费"、"招聘"、"论文"等）

## 输出规范
生成 Markdown 表格，包含列：
`关键词 | 匹配类型 | 搜索意图 | 购买阶段 | 广告组归属 | 优先级 | 备注`

如用户需要 Excel 文件，调用 xlsx-skill 生成可下载的 .xlsx 文件。

## 错误处理
- 若种子词过于宽泛（如"软件"），要求用户补充行业/功能细节
- 若词量超过 500，建议分批处理或优先聚焦 P1 词
```

# Google SEM 广告文案创作 Skill

## 概述
按照 Google RSA 规范，为指定广告组创作高质量广告文案，
覆盖标题库（≥10条）、描述库（≥4条），并标注固定/非固定建议。

## RSA 规格速查

| 元素 | 数量要求 | 字符限制 | 说明 |
|------|----------|----------|------|
| 标题（Headline） | 提供 ≥10 条 | 每条 ≤30 字符 | Google 动态组合展示 |
| 描述（Description） | 提供 ≥4 条 | 每条 ≤90 字符 | 每次展示选 2 条 |
| 展示网址路径 | 2 个路径字段 | 每个 ≤15 字符 | 需含核心关键词 |

## 核心工作流

### 步骤 1：信息收集
执行前确认（缺失则询问）：
- 广告组的核心关键词（2-5 个）
- 产品核心卖点（3-5 条）
- 独特价值主张（USP）
- CTA 偏好（免费试用 / 立即咨询 / 限时优惠 等）
- 品牌调性（专业 / 亲切 / 紧迫 等）

### 步骤 2：标题创作框架
按以下 5 个维度各创作 2-3 条标题：

| 维度 | 写法要点 | 示例（CRM软件） |
|------|----------|----------------|
| 关键词植入型 | 核心词前置，提升相关性 | "CRM客户管理软件" |
| 利益导向型 | 突出具体收益/数字 | "销售效率提升30%" |
| 痛点解决型 | 点名用户困扰 | "客户跟进不遗漏" |
| 社会证明型 | 权威背书/用户规模 | "10000+企业在用" |
| 紧迫感型 | 限时/限量/行动词 | "免费试用14天" |

### 步骤 3：描述语创作
每条描述语必须包含：
1. 核心功能或场景描述
2. 一个明确的 CTA（行动号召）
3. 可选：数字、权威认证、保障承诺

### 步骤 4：政策合规检查
读取 `references/policy-compliance.md`，逐条检查：
- [ ] 无最高级/绝对化词汇（"最佳"需有依据）
- [ ] 无不可核实的声明
- [ ] 标点使用规范（不超过 1 个感叹号）
- [ ] 大写规范（标题首字母大写，非全大写）
- [ ] 无竞品商标直接使用

### 步骤 5：固定位建议
标注哪些标题建议固定（Pin）到位置 1/2/3：
- 位置 1（必须固定）：包含核心关键词的标题
- 位置 2（建议固定）：核心 USP
- 位置 3（灵活）：CTA 或社会证明

## 输出规范

输出格式：

```
## 广告组：[广告组名称]
### 标题库（共 X 条）
1. [标题文案] （[字符数]/30）[建议: 固定至位置1 / 灵活]
2. ...

### 描述库（共 X 条）
1. [描述文案] （[字符数]/90）
2. ...

### 展示网址
域名/[路径1]/[路径2]

### 备注
- [政策风险提示（如有）]
- [A/B测试建议]
```

## 错误处理
- 字符超限：自动缩减并标注修改点
- 政策风险词：标红提示并提供替换建议
- 标题同质化：检测并要求补充差异化角度
```

# Google SEM 出价策略优化 Skill

## 概述
根据账户当前阶段（数据量、转化历史、预算规模），
推荐最适合的出价策略组合，并提供具体参数设置建议。

## 出价策略决策树

```
账户月转化量 < 30 次？
    │
    ├── 是 → 手动 CPC 或 目标展示次数份额
    │         重点：积累转化数据，不急于智能出价
    │
    └── 否 → 月转化量 30-100 次？
              │
              ├── 是 → 目标 CPA（保守目标，高于实际 CPA 20%）
              │
              └── 否（>100次）→ 目标 ROAS 或 最大化转化价值
                                  智能出价全面接管
```

## 核心工作流

### 步骤 1：账户数据诊断
收集（若用户未提供则询问）：
- 近 30 天转化次数
- 近 30 天实际 CPA
- 目标 CPA 或 目标 ROAS
- 月度预算上限
- 主要转化动作类型

### 步骤 2：策略匹配

| 策略 | 适用条件 | 优点 | 注意事项 |
|------|----------|------|----------|
| 手动 CPC | 新账户，数据不足 | 完全掌控 | 费时，靠人工经验 |
| 目标展示次数份额 | 品牌词保护 | 稳定曝光 | 不以转化为目标 |
| 目标 CPA | 月转化>30，有历史 CPA | 自动优化转化 | 目标设置需合理 |
| 目标 ROAS | 电商，转化有价值差异 | 以价值为导向 | 需准确传递转化价值 |
| 最大化转化量 | 预算有上限，求量 | 简单有效 | 可能 CPA 偏高 |
| 最大化转化价值 | 有预算上限，求价值 | ROI 导向 | 同上 |

### 步骤 3：参数计算建议

**目标 CPA 设置公式**：
```
首次设置目标 CPA = 历史实际 CPA × 1.2（给算法 20% 弹性空间）
稳定后每次调整幅度 ≤ 15%，调整后观察 ≥ 2 周
```

**目标 ROAS 设置公式**：
```
目标 ROAS = （目标利润率 + 1）× 100%
例：希望广告花 1 元带来 5 元收入 → ROAS = 500%
首次设置建议：历史实际 ROAS × 0.9（略低于实际值，避免流量受限）
```

### 步骤 4：出价调整建议（Bid Adjustments）
在智能出价基础上，建议以下维度的调整：

| 调整维度 | 建议逻辑 |
|----------|----------|
| 设备 | 移动端转化率若低于桌面端 30%+，考虑 -20% 移动出价 |
| 时段 | 分析转化时段分布，高转化时段 +10~20% |
| 地区 | 高价值城市或区域 +15~30% |
| 受众 | 再营销列表（RLSA）+30~50%；相似受众 +10~20% |

## 输出规范
输出包含：
1. 推荐策略及理由（2-3 句）
2. 具体参数设置值
3. 观察期建议（多少天后复查）
4. 预警指标（达到什么数字需要干预）
```

# Google SEM 质量得分诊断 Skill

## 概述
质量得分（QS）是 Google Ads 对广告质量的综合评分（1-10分），
直接影响广告排名（Ad Rank）和实际 CPC。
本 Skill 系统诊断 QS 的三个组成部分，给出优先级排序的优化方案。

## QS 三要素

| 组成部分 | 权重 | 诊断维度 |
|----------|------|----------|
| 预期点击率（eCTR） | 最高 | 文案吸引力、历史数据积累 |
| 广告相关性 | 中 | 关键词↔文案匹配度 |
| 落地页体验 | 中 | 相关性、加载速度、内容质量 |

## 核心工作流

### 步骤 1：症状识别
询问用户当前 QS 数字（如可提供）以及具体哪个子项标记为"低于平均"。

Google 在账户中展示三个等级：
- **高于平均**：该维度表现好，暂无需优化
- **平均**：可优化但非紧急
- **低于平均**：优先修复

### 步骤 2：按子项诊断

#### 预期点击率（eCTR）低于平均
根因清单：
- [ ] 文案未含关键词（相关性弱，用户不点）
- [ ] 标题缺乏 USP 或利益点
- [ ] 未使用广告附加功能（Sitelinks、Callout 等）
- [ ] 广告历史数据不足（新词/新文案，需积累时间）
- [ ] 竞品广告更具吸引力

修复动作：
1. 在标题 1 插入核心关键词
2. 补充数字、权威背书、限时信息
3. 添加至少 4 个 Sitelink 扩展
4. 测试新文案，给旧文案积累 2-4 周数据

#### 广告相关性低于平均
根因清单：
- [ ] 广告组过于宽泛，关键词覆盖主题不一致
- [ ] 文案未覆盖广告组主要关键词
- [ ] 使用了过于通用的文案模板

修复动作：
1. 细分广告组（将不同主题关键词分开）
2. 为每个广告组创作专属文案（关键词插入）
3. 使用 {KeyWord: 默认值} 动态关键词插入（DKI）

#### 落地页体验低于平均
根因清单：
- [ ] 落地页内容与广告关键词不匹配
- [ ] 页面加载速度慢（>3秒）
- [ ] 移动端体验差
- [ ] 内容稀薄，用户找不到想要的信息
- [ ] 过度弹窗/拦截导致用户流失

修复动作：
1. 落地页标题含广告关键词
2. 使用 Google PageSpeed Insights 测速，目标 LCP < 2.5s
3. 确保移动端完整展示核心信息（无需横向滚动）
4. 落地页与广告文案信息一致性检查

# Google SEM 账户结构规划 Skill

## 概述
一个好的账户结构是 SEM 长期效率的基础，影响出价控制、
数据分析粒度、文案相关性和预算分配灵活性。

## 标准三层架构

```
账户（Account）
└── 广告系列（Campaign）← 预算在这里控制
    └── 广告组（Ad Group）← 出价在这里控制
        ├── 关键词（Keywords）
        └── 广告（Ads）
```

## 广告系列分层原则

### 第一维度：流量类型隔离（必须分开）

| 系列类型 | 逻辑 | 原因 |
|----------|------|------|
| 品牌词系列 | 公司名/产品名相关词 | 高转化、低 CPC，不应与通用词混用 |
| 通用词系列 | 行业核心词 | 主要获客来源，预算最大 |
| 竞品词系列 | 竞争对手名称 | 独立预算控制，策略不同 |
| 长尾词系列 | 低量高意图词 | 单独优化，避免被高量词掩盖 |
| 再营销系列 | RLSA 受众叠加 | 竞价可更激进 |

### 第二维度：地区/语言（如适用）
- 不同投放地区 → 分开广告系列
- 不同语言 → 必须分开广告系列

### 第三维度：设备（高级）
- 移动端转化率差异 >30% → 考虑独立移动端系列

## 广告组策略选择

读取 `references/skag-vs-stag.md` 后根据以下框架决策：

| 场景 | 推荐策略 | 理由 |
|------|----------|------|
| 关键词量少（<50词）、品质要求高 | SKAG | 最高相关性，QS 最优 |
| 关键词量大（>200词）、资源有限 | STAG | 管理效率高 |
| 智能出价为主 | STAG + 广泛匹配 | 配合算法探索 |
| 手动出价，精细控制 | SKAG | 每词独立出价 |

## 输出规范
生成账户结构树形图（Markdown 格式）+ 预算分配建议表：

```
[账户名]
├── [品牌词-核心] 预算：¥XXX/天
│   ├── 广告组：品牌核心
│   └── 广告组：品牌+产品名
├── [通用词-核心功能] 预算：¥XXX/天
│   ├── 广告组：[功能词组1]
│   ├── 广告组：[功能词组2]
│   └── ...
└── [竞品词] 预算：¥XXX/天
    └── 广告组：[竞品名A]
```

# Google SEM 数据分析与报告 Skill

## 概述
接收 Google Ads 导出数据，执行结构化分析，
生成包含核心指标、趋势分析、异常诊断和优化建议的专业报告。

## 前置条件
- 数据时间范围（周/月/季度）
- 对比基准（上期/同比）
- 报告受众（内部优化 / 汇报给客户 / 汇报给管理层）

## 核心工作流

### 步骤 1：数据读取与清洗
读取上传文件（xlsx-skill 或 pdf-skill），识别以下核心指标列：

```python
# 关键指标映射
CORE_METRICS = {
    "展示次数": "Impressions",
    "点击次数": "Clicks",
    "点击率": "CTR",
    "平均CPC": "Avg. CPC",
    "费用": "Cost",
    "转化次数": "Conversions",
    "转化率": "Conv. Rate",
    "每次转化费用": "Cost/Conv.",
    "转化价值": "Conv. Value",
    "ROAS": "Conv. Value/Cost"
}
```

### 步骤 2：核心指标计算
| 指标 | 计算方式 | 健康参考范围（行业均值） |
|------|----------|----------------------|
| CTR | 点击/展示 | 搜索广告 3-7% |
| 转化率 | 转化/点击 | 行业差异大，2-5% 为一般水平 |
| 质量得分（加权均值） | Σ(QS×展示量)/总展示量 | ≥ 6 为健康 |
| 账户 ROAS | 转化价值/费用 | 视行业，通常 ≥ 300% |
| 无效点击占比 | 无效点击/总点击 | < 10% |

### 步骤 3：异常诊断
自动检测以下异常模式：

| 异常 | 诊断逻辑 | 可能原因 |
|------|----------|----------|
| CTR 环比下降 >20% | 同期对比 | 竞品增加、文案老化、季节性 |
| CPC 突然上涨 >30% | 同期对比 | 竞争加剧、质量得分下降 |
| 转化率下降但 CTR 正常 | 漏斗分析 | 落地页问题、受众漂移 |
| 展示量暴跌 | 绝对值对比 | 预算耗尽时间提前、竞价降低 |
| 某广告组 CPA >> 账户均值 | 账户内对比 | 需暂停或重构该广告组 |

### 步骤 4：报告生成
根据受众生成不同版本：

**内部优化版**：详细数据 + 根因分析 + 具体操作建议
**客户汇报版**：核心成果 + 趋势图 + 下期规划（淡化问题）
**管理层版**：投入产出比 + 与目标的差距 + 资源申请理由

## 输出规范
若需要正式文档，调用 docx-skill 或 xlsx-skill 生成可下载文件。
报告必须包含：
1. 执行摘要（3-5 个核心结论）
2. 关键指标总览表
3. 各广告系列/广告组对比
4. 异常原因分析
5. 下一步优化行动（带优先级）
```

# Google SEM 落地页质检 Skill

## 概述
从 SEM 三个维度评估落地页：相关性（影响 QS）、
转化优化（影响 CVR）、政策合规（影响账户安全）。

## 落地页 SEM 质检清单

### 维度 1：广告相关性（影响 Quality Score）
- [ ] 落地页 H1 标题含广告核心关键词
- [ ] 页面 Title Tag 含关键词
- [ ] 首屏内容与广告承诺一致（用户点广告后见到的内容匹配）
- [ ] 无"广告说 A，落地页讲 B"的信息断层
- [ ] URL 路径含关键词（如 /crm-software/）

### 维度 2：转化率优化（CRO 基础检查）
- [ ] 首屏有清晰的价值主张（5 秒原则）
- [ ] CTA 按钮在首屏可见，颜色对比明显
- [ ] 表单字段 ≤ 5 个（每增加一个字段降低约 10% 转化）
- [ ] 有社会证明（客户评价、案例、数字等）
- [ ] 移动端一键拨打电话按钮（如适用）
- [ ] 页面加载时间 < 3 秒（建议用 PageSpeed Insights 测速）

### 维度 3：Google Ads 政策合规
- [ ] 无弹窗拦截用户立即离开页面
- [ ] 联系方式/公司信息可见
- [ ] 无误导性内容（与广告描述完全一致）
- [ ] 隐私政策页面可访问
- [ ] 无恶意软件/不寻常的重定向

## 输出规范
输出分级诊断报告：
- 🔴 必须修复（影响账户投放）
- 🟡 建议优化（影响转化和QS）
- 🟢 锦上添花（进阶优化建议）

每项问题附：当前状况描述 + 修复建议 + 预期改善效果。
