## 前言

SEO 工作高度依赖**重复性流程** + **专业判断**的组合——这正是 Skill 体系最擅长处理的场景。

本文基于 [AI Skill 体系设计哲学](../how-to-build-skill-system/)，为 SEO 从业者和开发者提供一套可直接部署到 Claude Code 的 **SEO Skill 合集**，涵盖：

- 🔍 关键词研究与意图分析
- 📝 内容 SEO 优化
- 🔧 技术 SEO 审计
- 🔗 外链机会挖掘
- 📊 SEO 报告生成

每个 Skill 均提供完整的 `SKILL.md` 模板和 Claude Code Prompt。

### SKILL.md 模板

````markdown
---
name: seo-keyword-research
description: |
  执行专业的 SEO 关键词研究与搜索意图分析。当用户提到关键词挖掘、
  词库扩展、搜索量分析、竞争度分析、长尾词发现、用户搜索意图分类时
  必须触发此技能。即使用户只说"帮我找关键词"或"分析一下搜索词"也应触发。

compatibility:
  tools: [web_search, bash]
---

# 关键词研究 Skill

## 工作流

### 步骤 1：确认研究范围
- 目标行业 / 产品类别
- 核心种子词（1-5 个）
- 地区和语言（默认：中文简体，中国大陆）
- 目标页面类型（首页 / 产品页 / 博客 / 落地页）

### 步骤 2：关键词扩展矩阵

按以下四象限扩展：

| 类型 | 特征 | 示例 |
|------|------|------|
| 品牌词 | 包含品牌名 | "品牌名 + 功能" |
| 产品词 | 直接描述产品 | "XX 软件购买" |
| 信息词 | 用户学习阶段 | "XX 是什么" / "如何使用XX" |
| 竞品词 | 竞争对手相关 | "XX 替代品" / "XX vs YY" |

### 步骤 3：搜索意图分类（TOFU/MOFU/BOFU）

- **TOFU（漏斗顶部）**：泛信息词，触达潜在用户
- **MOFU（漏斗中部）**：方案对比词，影响考量决策
- **BOFU（漏斗底部）**：购买意图词，驱动转化

### 步骤 4：竞争难度评估

对每个关键词评估：
- 搜索意图明确度（1-5 分）
- 内容产出可行性（是否有独特价值可提供）
- 建议优先级（高/中/低）

### 步骤 5：输出关键词报告

生成结构化 Markdown 表格，包含：
- 关键词 | 意图类型 | 漏斗阶段 | 优先级 | 建议内容形式

## 输出规范

- 输出为 Markdown 表格 + 简要策略说明
- 关键词数量：种子词 ×10 扩展（每类至少 3 个）
- 必须包含：高优先级关键词的内容建议
````

````markdown
# 内容 SEO 优化 Skill

## 前置条件

获取以下信息后再开始优化：
- [ ] 目标关键词（主词 + 2-3 个相关词）
- [ ] 目标读者画像
- [ ] 当前文章内容（粘贴或上传）
- [ ] 竞争对手 URL（可选，用于差异化分析）

## 核心工作流

### 步骤 1：SEO 健康诊断

检查以下 12 项指标并打分（✅/⚠️/❌）：

**标题优化层**
- [ ] H1 标签包含主关键词
- [ ] H1 长度 20-60 字符
- [ ] H2/H3 覆盖相关词和问题词

**Meta 信息层**
- [ ] Title Tag：含主词，50-60 字符
- [ ] Meta Description：含行动召唤，150-160 字符

**内容质量层**
- [ ] 关键词自然分布（首段、小标题、结尾）
- [ ] 内容深度（字数是否达到竞争水平）
- [ ] 原创性和独特价值点
- [ ] FAQ 段落（覆盖 People Also Ask）

**内部链接层**
- [ ] 是否有 3+ 个内链锚文本机会
- [ ] 图片 Alt 标签优化
- [ ] 结构化数据建议

### 步骤 2：输出优化版本

按优先级输出：
1. **必改**：标题、Meta、H2 结构
2. **建议**：段落改写、FAQ 添加
3. **可选**：内链建议、Schema 标记

## 输出规范

每项优化提供：原文 → 优化版本 + 改动理由
````

## 三、技术 SEO 审计 Skill

### SKILL.md 模板

````markdown
---
name: seo-technical-audit
description: |
  执行网站技术 SEO 全面审计。当用户提到技术SEO检查、网站爬取问题、
  Core Web Vitals、页面速度优化、robots.txt、sitemap、
  规范化URL（canonical）、结构化数据、移动端适配、HTTPS 安全性、
  重定向链问题时必须触发此技能。用户说"帮我查一下网站技术问题"也应触发。
compatibility:
  tools: [web_search, web_fetch, bash]
---

# 技术 SEO 审计 Skill

## 审计范围选择

根据用户输入决定审计深度：

| 场景 | 审计类型 | 重点 |
|------|----------|------|
| 新站上线前 | 基础检查 | robots/sitemap/HTTPS/索引 |
| 流量下降排查 | 深度诊断 | 爬取/重定向/规范化/惩罚 |
| 全站健康检查 | 完整审计 | 所有技术指标 |
| 性能优化 | 速度专项 | CWV/图片/JS/CSS |

## 核心工作流

### 步骤 1：可索引性检查
- 抓取 robots.txt，检查是否误封重要目录
- 验证 XML Sitemap 格式和更新时效
- 检查 noindex/nofollow 使用是否合理
- 确认 canonical 标签指向正确

### 步骤 2：技术健康指标
- HTTPS 部署（混合内容问题）
- 移动端适配（viewport/响应式）
- 页面加载速度指标（CWV：LCP/FID/CLS）
- 重定向链（超过 2 跳需处理）
- 404 错误页面处理

### 步骤 3：结构化数据验证
- 检查 Schema.org 标记类型
- 验证 JSON-LD 格式正确性
- 推荐适合该站类型的 Schema

### 步骤 4：输出审计报告

按严重程度分类：
- 🔴 **紧急修复**（影响索引或排名）
- 🟡 **优化建议**（影响用户体验）
- 🟢 **最佳实践**（锦上添花）

每项问题提供：问题描述 → 影响范围 → 修复方案 → 优先级

## 输出规范

生成结构化审计报告，包含：执行摘要、问题清单、修复路线图
````

```
- SEO 影响说明
- 具体修复方案（含代码示例）
- 预估修复工时

**第五步：修复优先级路线图**
按 [影响度 × 修复难度] 矩阵，输出 30天 / 60天 / 90天 修复计划。
```

````markdown
# 外链机会挖掘 Skill

## 核心工作流

### 步骤 1：竞品外链格局分析

通过搜索确定：
- 行业内高 DA（域名权重）的参考站点
- 竞品普遍拥有但我方缺失的链接类型
- 本行业常见的外链获取渠道

### 步骤 2：外链机会分类

| 机会类型 | 获取难度 | 质量 | 适用场景 |
|----------|----------|------|----------|
| 资源页链接 | 中 | 高 | 工具类/指南类内容 |
| 嘉宾博客 | 高 | 高 | 有专家背书的品牌 |
| 品牌提及追踪 | 低 | 中 | 已有媒体曝光的品牌 |
| 损坏链接替换 | 中 | 高 | 有高质量替代内容 |
| 行业目录收录 | 低 | 中低 | 新站快速建立基础 |
| 数据/研究引用 | 高 | 极高 | 有原创数据的内容 |

### 步骤 3：推广材料准备建议

针对目标外链类型，建议准备：
- 外联邮件模板（个性化变量清单）
- 价值主张清单（为什么对方应该链接你）
- 目标站点筛选标准（DA阈值/行业相关性/流量要求）

### 步骤 4：输出外链策略报告

包含：机会列表、优先级排序、行动计划
````

````markdown
# SEO 报告生成 Skill

## 报告类型路由

根据用户描述，选择对应报告模板：

- 提到"月报"/"季报" → 定期 SEO 绩效报告
- 提到"竞品" → 竞品 SEO 分析报告
- 提到"流量下降"/"诊断" → SEO 问题诊断报告
- 提到"新站"/"上线" → SEO 启动评估报告

## 定期绩效报告工作流

### 核心模块

1. **执行摘要**：3 句话总结本期核心成果
2. **核心指标看板**：自然流量、关键词排名、点击率、转化
3. **关键词表现**：上升词 TOP10 / 下降词 TOP10 / 新进词
4. **内容表现**：流量增长最多的页面 + 原因分析
5. **技术健康**：本期修复问题 + 待处理问题
6. **外链动态**：新增外链 / 丢失外链
7. **下期计划**：3 个优先行动项（含预期效果）

## 输出规范

- 格式：Markdown（可选输出 .docx）
- 长度：执行摘要 1 页 + 详细分析 3-5 页
- 数据可视化：用 ASCII 图表或 Markdown 表格替代
- 受众适配：可选"技术版"（含细节）或"管理层版"（含结论）
````
