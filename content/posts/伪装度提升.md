在使用 Whoer.net 或 Pixelscan 等工具检测时，伪装度显示为 90% 主要是因为本地设置与 IP 所在地不一致导致的。这 10% 的差距往往就是导致领英、Facebook 注册即风控的“临门一脚”。
以下是把伪装度从 90% 提升到 100% 的具体处理步骤：
## 1. 时区不一致 (Timezone) —— 最常见原因
这是 90% 变成 100% 最核心的扣分项。
 * 问题： 你的 IP 所在地是洛杉矶（西八区），但你手机或电脑系统的时区还是北京时间（东八区）。
 * 解决： * 电脑端： 进入系统设置，关闭“自动设置时间”，手动将时区修改为与 IP 所在地一致（例如：UTC-08:00）。
   * 指纹浏览器： 在 AdsPower 或 HubStudio 浏览器环境设置里，勾选 “基于 IP 启用扫描检测时区”，它会自动同步。
## 2. 语言设置不匹配 (Language)
 * 问题： IP 在美国，但浏览器或系统的第一语言是“中文”。
 * 解决：
   * 电脑/手机系统： 将系统语言改为英语（English - US）。
   * 浏览器： 在设置中将“英语”排在语言偏好的第一位，并删除中文。
   * 指纹浏览器： 在环境配置里开启“语言基于 IP 匹配”。
## 3. DNS 泄露 (DNS Leak)
 * 问题： 你的 IP 显示在美国，但 DNS 服务器却显示在中国（通常是 114.114.114.114 或运营商 DNS）。
 * 解决：
   * 如果你用的是指纹浏览器，确保在代理设置里开启了“远程 DNS 解析”。
   * 如果你用的是软路由/代理软件，开启“全局模式”并勾选“禁止 DNS 泄露”或使用类似 8.8.8.8 的国际通用 DNS。
## 4. 系统时间分钟误差
 * 问题： 时区对上了，但你的系统时间比标准时间快或慢了几分钟。
 * 解决： 在修改完时区后，点击系统设置里的“立即同步”，确保秒级同步。
## 5. WebRTC 泄露
 * 问题： WebRTC 是一种用于网页实时通信的协议，它会绕过代理直接暴露你的真实局域网 IP。
 * 解决：
   * 电脑端： 安装 Chrome 插件 WebRTC Leak Prevent。
   * 指纹浏览器： 在设置中将 WebRTC 选项设为 “替换 (Replace)” 或 “禁用 (Disabled)”。
针对你的钢材外贸场景，特别检查：
如果你是在手机上操作：
 * 黑名单检查： 检查 Whoer 里的“Blacklist”这一项。如果 IP 在黑名单，即使 100% 也会被风控。这时候必须换 IP。
 * 经纬度： 手机操作时，如果 IP 在纽约，但 GPS 定位在中国，伪装度会大幅下降。必须彻底关闭手机的定位服务。
总结操作流程：
 * 打开 Whoer.net，点击中间的“百分比”图标，它会明确列出扣分项（红字）。
 * 如果是 Timezone，就去改系统时区。
 * 如果是 Language，就去改语言设置。
 * 修改完后，清除浏览器缓存（或在指纹浏览器重启环境），重新刷新页面。
达到 100% 之后再打开领英官网进行注册操作，这样被判定为“真人”的概率会提升 80% 以上。
