## 什么是博客支柱页？

支柱页（Pillar Page）是内容集群策略的核心页面，负责全面覆盖某一主题，并通过内链指向多篇深度子话题文章（Cluster Content）。良好的支柱页结构能显著提升站点权威度与搜索排名。

---

## 需要注册新页面吗？

**不一定需要。** WordPress 本身已提供多种机制可直接实现支柱页，无需开发自定义 Post Type 或注册额外页面类型。根据站点结构和需求，有以下三种主流方案：

---

## 方案一：使用普通 Page（推荐新手）

直接在 **页面 → 新建页面** 中创建支柱页，手动编写完整的主题概述，并在正文中插入内链指向相关博客文章。

**优点：**
- 零开发成本，操作最简单
- URL 结构清晰（如 `/topic/seo-guide/`）
- 与 Gutenberg 完全兼容，可使用块编辑器排版

**缺点：**
- 内链需手动维护，子文章增多后管理成本上升
- 无法自动聚合相关文章列表

**适合场景：** 内容量较少、更新频率低的站点。

---

## 方案二：使用 Category 分类存档页（推荐内容站）

将某一核心主题设为独立分类（Category），并通过自定义分类模板（`category-{slug}.php`）将其改造为支柱页样式。

**实现思路：**

```php
// 在主题目录创建 category-seo-guide.php
// 在顶部加入自定义介绍区块，下方自动循环输出该分类所有文章
```

**优点：**
- 子文章归入该分类后**自动聚合**，无需手动维护内链列表
- SEO 天然继承分类权重
- URL 可配置为 `/blog/seo-guide/`

**缺点：**
- 需要具备一定主题开发能力
- 依赖固定链接设置

**适合场景：** 内容持续产出、需要自动化管理的博客站。

---

## 方案三：使用自定义 Post Type + ACF（推荐企业站）

通过 **Custom Post Type UI** 插件注册 `pillar_page` 自定义类型，配合 **Advanced Custom Fields（ACF）** 管理支柱页与子文章的关联关系。

**实现思路：**

```
注册 CPT: pillar_page
↓
ACF 字段：关联文章（Relationship 字段，指向 post）
↓
模板文件：single-pillar_page.php 循环输出关联文章卡片
```

**优点：**
- 内容结构最清晰，支柱页与普通文章完全解耦
- 支持在后台可视化管理关联关系
- 便于多人协作的内容团队使用

**缺点：**
- 配置成本最高
- 依赖第三方插件

**适合场景：** 内容团队较大、需要系统化内容运营的企业级站点。

---

## 三种方案对比

| 方案 | 开发成本 | 自动化程度 | 维护成本 | 适合场景 |
|------|----------|------------|----------|----------|
| 普通 Page | 低 | 手动 | 高 | 小型站点 |
| Category 存档页 | 中 | 自动 | 低 | 内容博客 |
| CPT + ACF | 高 | 自动 | 低 | 企业站 |

---

## 建议

对于大多数 WordPress 博客站，**方案二（Category 存档页）** 是性价比最高的选择：无需额外注册新页面类型，利用 WordPress 原生分类机制即可实现内容自动聚合，同时保持良好的 URL 结构和 SEO 友好性。

如果站点处于起步阶段，先用**方案一**快速落地，待内容量增长后再迁移至方案二，是常见的渐进式策略。
