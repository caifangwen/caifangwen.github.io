## 为什么需要一套文件整理工作流？

你的桌面是否堆满了随手放置的文件？下载文件夹是否已经成了"黑洞"？每次找文件都要翻找好几分钟？

这些问题的根源不是懒惰，而是**缺少一套能自动运转的工作流**。手动整理文件的问题在于它依赖意志力——你必须在每次下载、保存、接收文件后主动去分类。而工作流的本质是把这套判断逻辑**外包给工具**，让 DropIt 代替你做决策。

本文将完整讲解如何用 DropIt 搭建一套覆盖日常所有场景的文件整理工作流。

---

## 工作流总体设计思路

在动手配置之前，先设计好整体框架。一套好的文件整理工作流需要回答三个问题：

### 1. 文件从哪里来？（输入源）

| 输入源 | 说明 |
|--------|------|
| 浏览器下载文件夹 | 最主要的文件入口 |
| 桌面 | 临时存放的截图、文件 |
| 邮件附件保存目录 | Outlook / 邮件客户端的默认保存位置 |
| 手机同步文件夹 | 照片、录音等手机传输的内容 |
| 工作协作目录 | 共享盘、云盘本地同步文件夹 |

### 2. 文件应该去哪里？（目标结构）

推荐按**用途**而非类型来设计顶层目录结构：

```
D:\Files\
├── Inbox\          ← 临时中转站（所有未分类文件先到这里）
├── Work\           ← 工作相关
│   ├── Projects\   ← 按项目存放
│   ├── Reports\    ← 报告、汇报材料
│   └── Reference\  ← 参考资料
├── Personal\       ← 个人事务
│   ├── Finance\    ← 账单、合同
│   └── Health\     ← 医疗记录
├── Media\          ← 媒体文件
│   ├── Photos\     ← 按年月分类的照片
│   ├── Videos\     ← 视频
│   └── Music\      ← 音乐
├── Archive\        ← 归档（不常用但需保留的文件）
│   └── 2025\
└── Trash\          ← 软删除缓冲区（正式删除前先放这里）
```

### 3. 由什么触发整理？（触发方式）

DropIt 支持两种触发方式，应结合使用：

| 触发方式 | 适用场景 | 优点 |
|----------|----------|------|
| **手动拖放** | 处理单个或少量文件 | 灵活，可即时预览 |
| **文件夹监控** | 下载目录、桌面等高频输入源 | 全自动，无需干预 |

## 工作流一：下载文件夹自动分流

这是使用频率最高、收益最大的工作流。

### 目标

将 `C:\Users\你的用户名\Downloads` 中的文件，按类型自动分发到对应目录，并在 30 秒内完成，无需手动操作。

### 规则配置

在 DropIt 中新建配置文件，命名为 **"下载整理"**，依次添加以下规则（顺序即优先级）：

#### 规则 1：工作文档

- **名称**：工作文档
- **过滤器**：`*.pdf;*.docx;*.doc;*.xlsx;*.xls;*.pptx;*.ppt`
- **附加条件**：文件大小 > 0 KB（排除空文件）
- **动作**：移动
- **目标路径**：`D:\Files\Work\Reference\`

#### 规则 2：图片文件

- **名称**：图片
- **过滤器**：`*.jpg;*.jpeg;*.png;*.gif;*.webp;*.bmp;*.svg;*.heic`
- **动作**：移动
- **目标路径**：`D:\Files\Media\Photos\Downloads\`

#### 规则 3：视频文件

- **名称**：视频
- **过滤器**：`*.mp4;*.mkv;*.avi;*.mov;*.wmv;*.flv;*.webm`
- **动作**：移动
- **目标路径**：`D:\Files\Media\Videos\`

#### 规则 4：音频文件

- **名称**：音频
- **过滤器**：`*.mp3;*.flac;*.wav;*.aac;*.ogg;*.m4a`
- **动作**：移动
- **目标路径**：`D:\Files\Media\Music\`

#### 规则 5：压缩包

- **名称**：压缩包
- **过滤器**：`*.zip;*.rar;*.7z;*.tar;*.gz;*.iso`
- **动作**：移动
- **目标路径**：`D:\Files\Inbox\Archives\`

#### 规则 6：安装程序（特殊处理）

- **名称**：安装程序
- **过滤器**：`*.exe;*.msi;*.dmg;*.pkg`
- **动作**：移动
- **目标路径**：`D:\Files\Inbox\Setup\`

> 安装程序建议单独存放，安装完成后再手动清理，不要直接删除，方便重装。

#### 规则 7：代码与文本文件

- **名称**：代码文本
- **过滤器**：`*.py;*.js;*.ts;*.html;*.css;*.json;*.xml;*.csv;*.txt;*.md`
- **动作**：移动
- **目标路径**：`D:\Files\Work\Reference\Code\`

#### 规则 8：账单财务类 PDF（需放在规则 1 之前）

- **名称**：账单财务
- **过滤器**：`账单*;*发票*;*收据*;*合同*;*对账*`
- **动作**：移动
- **目标路径**：`D:\Files\Personal\Finance\[year]\`

> ⚠️ 此规则需放在"工作文档"规则**之前**，否则带有 `.pdf` 扩展名的发票会先被规则 1 捕获。

### 关联文件夹设置

1. 进入 **关联和规则 → 文件夹** 标签页
2. 添加 `C:\Users\你的用户名\Downloads`
3. 绑定配置文件：**"下载整理"**
4. 监控延迟：**60 秒**（等待浏览器完成下载）
5. 勾选：**监控子文件夹**（可选）
6. 勾选：**Windows 启动时自动激活**

完成后，下载文件夹将实现全自动分流，新文件在下载完成 60 秒后自动移走。

## 工作流二：桌面清洁工作流

桌面是第二大文件堆积地。这套工作流的核心思路是：**桌面只是临时停靠，不是永久存储**。

### 目标

每天下班前或每隔一段时间，自动清理桌面上的文件到对应位置，保持桌面整洁。

### 规则配置

新建配置文件，命名为 **"桌面清洁"**：

#### 规则 1：截图文件（识别截图命名规律）

- **过滤器**：`截图*;Screenshot*;屏幕截图*;Snipaste*;*screenshot*`
- **动作**：移动
- **目标路径**：`D:\Files\Media\Photos\Screenshots\[year]\[month]\`

#### 规则 2：快捷方式（保留不动）

- **过滤器**：`*.lnk`
- **动作**：忽略

#### 规则 3：临时文本记录

- **过滤器**：`*.txt;*.md`
- **动作**：移动
- **目标路径**：`D:\Files\Inbox\Notes\`

#### 规则 4：其余所有文件（兜底规则）

- **过滤器**：`*`
- **附加条件**：修改日期早于 **3 天**
- **动作**：移动
- **目标路径**：`D:\Files\Inbox\`

> 兜底规则只处理 3 天前的文件，给当天和最近工作中的文件留出缓冲时间。

### 触发方式

桌面清洁不建议设置实时监控（防止正在使用的文件被移走），推荐两种方式：

**方式 A：定时触发（推荐）**

在 Windows 任务计划程序中创建定时任务，每天 18:30 触发 DropIt 对桌面执行整理。

**方式 B：手动拖放触发**

选中桌面上需要整理的文件，直接拖到 DropIt 悬浮图标上，即时处理。

## 工作流三：照片归档工作流

照片管理的难点在于**数量多、来源杂、时间跨度长**。这套工作流利用 DropIt 读取 EXIF 信息的能力，按拍摄时间自动归档。

### 目标结构

```
D:\Files\Media\Photos\
├── 2024\
│   ├── 01\
│   ├── 02\
│   └── ...
├── 2025\
│   ├── 08\
│   └── ...
└── NoDate\     ← 无 EXIF 信息的图片
```

### 规则配置

新建配置文件，命名为 **"照片归档"**：

#### 规则 1：带 EXIF 的相机照片

- **过滤器**：`*.jpg;*.jpeg;*.heic;*.dng;*.raw;*.cr2;*.nef`
- **动作**：移动
- **目标路径**：`D:\Files\Media\Photos\[exif:year]\[exif:month]\`

> `[exif:year]` 和 `[exif:month]` 会读取照片的实际拍摄时间，而非文件的修改时间，确保归档准确。

#### 规则 2：截图和无 EXIF 的 PNG

- **过滤器**：`*.png;*.webp;*.gif`
- **动作**：移动
- **目标路径**：`D:\Files\Media\Photos\NoDate\`

### 使用方式

将手机导出的照片文件夹整体拖放到 DropIt 悬浮图标上，DropIt 会递归处理所有子目录中的照片，按拍摄时间一次性归档完毕。

## 工作流四：项目文件生命周期管理

工作项目文件有明确的生命周期：**进行中 → 已完结 → 归档**。这套工作流管理项目文件在不同阶段之间的流转。

### 目录结构设计

```
D:\Files\Work\
├── Active\         ← 当前正在进行的项目
│   ├── ProjectA\
│   └── ProjectB\
├── Done\           ← 已完结、本年度暂不归档的项目
└── Archive\        ← 按年份归档的历史项目
    ├── 2024\
    └── 2025\
```

### 规则配置

新建配置文件，命名为 **"项目归档"**：

#### 规则 1：归档已完结项目文件夹

- **过滤器**：`*`（匹配文件夹）
- **类型**：仅文件夹
- **动作**：移动
- **源路径**：`D:\Files\Work\Done\`
- **目标路径**：`D:\Files\Work\Archive\[year]\`

#### 规则 2：清理项目中的临时文件

- **过滤器**：`*.tmp;~$*;*.bak;Thumbs.db`
- **动作**：删除
- **说明**：`~$*` 匹配 Office 程序产生的临时锁定文件

### 使用方式

项目完结后：

1. 将项目文件夹从 `Active\` 手动移到 `Done\`
2. 每月末，将 `Done\` 文件夹拖放到 DropIt，自动按年份归入 `Archive\`

这样既保留了手动确认的控制感，又让归档动作本身变得一键完成。

## 工作流五：定期清理与软删除工作流

直接删除有风险，但"先移到垃圾桶"的软删除策略更安全。

### 目标

对 `Inbox\`（中转站）中超过 30 天未处理的文件，自动移入 `Trash\` 软删除区，再由另一条规则在 60 天后彻底删除。

### 规则配置

新建配置文件，命名为 **"定期清理"**：

#### 规则 1：将过期 Inbox 文件移入软删除区

- **过滤器**：`*`
- **附加条件**：修改日期早于 **30 天**
- **源监控目录**：`D:\Files\Inbox\`
- **动作**：移动
- **目标路径**：`D:\Files\Trash\`

#### 规则 2：彻底删除软删除区的过期文件

- **过滤器**：`*`
- **附加条件**：修改日期早于 **60 天**
- **源监控目录**：`D:\Files\Trash\`
- **动作**：删除

### 执行方式

通过 Windows 任务计划程序，设置每周日晚 22:00 自动对 `Inbox\` 和 `Trash\` 执行一次规则扫描。

## 多配置文件的切换与管理

以上五套工作流，在 DropIt 中对应五个配置文件。日常管理建议如下：

| 配置文件 | 触发方式 | 频率 |
|----------|----------|------|
| 下载整理 | 文件夹自动监控 | 实时（60 秒延迟） |
| 桌面清洁 | 任务计划 / 手动 | 每日 |
| 照片归档 | 手动拖放 | 手机导出时 |
| 项目归档 | 手动拖放 | 每月末 |
| 定期清理 | 任务计划 | 每周 |

右键 DropIt 悬浮图标，在配置文件菜单中一键切换当前激活的配置文件，对应的监控和规则随之生效。

## 规则调试与安全建议

### 新规则上线前：先用"忽略"动作测试

在规则完全上线前，将动作改为 **"忽略（Ignore）"**，然后拖放一批测试文件，观察 DropIt 的匹配日志，确认规则覆盖范围符合预期后，再改为实际动作。

### 移动优先于删除

所有自动规则，**能用"移动"就不用"删除"**。先移动到 `Trash\` 软删除区，保留一段时间后再清理，为误操作留出挽回余地。

### 重要目录加入排除列表

在监控设置中，将以下目录**排除在监控之外**，防止意外处理：

- `C:\Windows\`
- `C:\Program Files\`
- 任何正在使用中的项目目录

### 定期导出备份配置

所有规则和配置文件导出为 `.dpi` 文件，存放在云盘，防止重装系统后丢失：

**右键图标 → 导入/导出 → 导出所有配置**

## 小结

一套完整的 DropIt 工作流，本质上是把你脑海中"这个文件该放哪"的决策逻辑，提前固化成规则，让工具替你持续执行。

搭建这套体系的核心步骤：

1. **设计目录结构** — 按用途而非类型划分，层级不超过三层
2. **识别主要输入源** — 下载、桌面、手机、邮件
3. **配置对应规则** — 精确覆盖，兜底规则兜住剩余情况
4. **设置自动触发** — 高频入口用监控，低频操作用任务计划
5. **软删除保底** — 所有自动删除前必须经过缓冲区

初次搭建可能需要 1-2 小时，但一旦运行起来，你的文件系统将几乎不再需要手动干预。
