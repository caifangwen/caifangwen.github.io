## 什么是 Claude Cowork？

长久以来，AI 助手只能活在对话框里——你问一句，它答一句，文件还得自己上传，格式还得自己整理。**Claude Cowork** 打破了这个局限。

Cowork 是 Anthropic 于 2026 年 1 月推出的桌面自主代理（Agentic）工具，内置于 Claude 桌面应用程序，与 Chat、Claude Code 并列为三大核心模式之一。它将 Claude Code 的"自主规划、多步执行"能力，以图形界面的形式带给所有知识工作者——不需要打开终端，不需要懂编程，只需描述你想要的结果，然后去做别的事，回来取成果就好。

> **一句话定义**：Cowork = Claude Code 的自主执行力 + 普通用户能直接上手的桌面界面。

---

## 核心能力

### 1. 直接读写本地文件

在普通对话模式下，Claude 只能分析你上传的文件，无法直接触达你的电脑。Cowork 则不同——你授权它访问指定文件夹后，它可以直接**读取、编辑、新建**本地文件，真正完成任务，而不是告诉你"你应该怎么做"。

支持的文件格式涵盖：Word 文档（`.docx`）、PDF、Excel 电子表格（`.xlsx`）、PowerPoint 演示文稿（`.pptx`）、Markdown、CSV、JSON、HTML、纯文本等主流格式。

### 2. 自主规划与多步执行

Cowork 接到任务后，会自动分析、制定计划，将复杂任务拆解为多个子任务，甚至**并行协调多个子代理**同时推进，并在过程中持续向你汇报进度。整个流程不需要你一步一步地手动推进。

### 3. 生成专业文档输出

Cowork 内置了针对办公文件格式的专项"技能"（Skills），能够生成带有正确公式的 Excel 电子表格、带有图表与排版的 PowerPoint 演示文稿、格式规范的 Word 报告等专业级交付物。

### 4. 连接外部工具（Connectors & MCP）

通过连接器（Connectors）和模型上下文协议（MCP），Cowork 可以与 Slack、Notion、Figma、Google Workspace 等外部服务对接，将跨平台的信息汇聚在一个任务流程中完成。配合 **Claude in Chrome**，还可以执行需要浏览器操作的任务。

### 5. 定时与计划任务

Cowork 支持**定时任务**（Scheduled Tasks），可以配置每日简报、每周报告、定期文件处理等自动化工作流。Claude 按计划在后台运行，你无需在场。

### 6. 移动端远程下达任务（Dispatch）

通过手机上的 Claude App，你可以随时向桌面端的 Cowork 下达任务。Claude 在你的桌面电脑上调用本地文件和已配置的连接器完成工作，并将结果发回给你——即便你不在电脑前。

### 7. 项目管理（Projects）

2026 年 3 月 20 日，Cowork 正式推出了 **Projects** 功能，允许用户将相关任务、文件、指令和上下文组织在独立的工作区中，保持跨会话的任务连续性，彻底告别每次都要重新交代背景的烦恼。

## 典型使用场景

| 场景 | 描述 |
|------|------|
| **文件整理** | 指向一个混乱的下载文件夹，Claude 自动分类、重命名、清理，并在动手前给你确认计划 |

| **定期自动化** | 每周一自动生成周报、每月汇总费用报表，Claude 按计划自行执行 |

## 与普通 Claude Chat 的区别

|  | Claude Chat | Claude Cowork |
|--|-------------|---------------|
| 文件访问 | 需手动上传，单次有限制 | 直接读写授权的本地文件夹 |
| 任务模式 | 一问一答 | 自主规划、多步执行 |
| 是否需要在场 | 是 | 否，可以离开等结果 |
| 输出形式 | 文字回复 | 直接生成/修改本地文件 |
| 外部集成 | 有限 | 支持 MCP、Connectors |
| 定时任务 | 不支持 | 支持 |

## 平台支持与订阅要求

Cowork 目前处于**研究预览（Research Preview）**阶段，持续迭代更新。

- **平台**：macOS 与 Windows（需最新版 Claude 桌面应用，**不支持** Windows arm64）
- **订阅**：仅限付费计划（Pro、Max、Team、Enterprise）
- **注意**：桌面应用必须保持开启状态，Cowork 才能正常执行任务；会话历史存储在本地，不计入 Anthropic 数据保留周期。

---

## 安全与权限控制

Cowork 以"你掌握控制权"为设计原则：

- Claude 只能访问你**明确授权**的文件夹和连接器
- 在执行重要操作前（如删除文件），Claude 会**主动询问确认**
- 代码在**隔离的虚拟机（VM）环境**中运行，网络访问通过白名单控制
- 会话历史保存在本地，不上传云端，不出现在审计日志中

---

## 当前局限

作为研究预览版本，Cowork 目前尚不支持：

- 跨设备会话同步
- Artifacts 功能
- Google 集成（Drive、Docs 等）
- 跨会话的 Memory（记忆）

Anthropic 已明确表示后续版本将逐步补齐这些能力。
