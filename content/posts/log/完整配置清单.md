# Hugo Narrow 主题配置清单

## ✅ 必须配置的项目

### 1. Giscus 评论系统（已启用但需配置）

#### 步骤 1：创建 GitHub 仓库
```
仓库名建议: blog-comments
设置: 公开仓库
```

#### 步骤 2：安装 Giscus App
访问: https://github.com/apps/giscus
点击 "Install" 并选择刚创建的仓库

#### 步骤 3：启用 Discussions
在仓库设置中：
Settings → Features → ✅ Discussions

#### 步骤 4：获取配置信息
访问: https://giscus.app/zh-CN

填写：
- 仓库: `caifangwen/blog-comments`
- 页面 ↔️ discussion 映射: pathname
- Discussion 分类: Announcements（推荐）
- 特性: ✅ 启用反应

复制生成的配置信息

#### 步骤 5：修改 params.yaml
```yaml
comments:
  enabled: true
  system: "giscus"
  giscus:
    repo: "caifangwen/blog-comments"  # 你的仓库
    repoId: "R_xxx"  # 从 giscus.app 复制
    category: "Announcements"
    categoryId: "DIC_xxx"  # 从 giscus.app 复制
    lang: "zh-CN"
```

```css

@media (max-width: 768px) {
  body.home .post-list article > a > div > div:first-child {
    display: none !important;
  }
  body.home .post-list article > a > div {
    padding: 0.875rem 1rem !important;
  }
  body.home .post-list article h3 {
    margin-bottom: 0.5rem !important;
  }
  body.home .post-list article p {
    margin-bottom: 0.5rem !important;
  }
  body.home .post-list .space-y-4 > * + * {
    margin-top: 0.5rem !important;
  }
}
```

**确保 params.yaml 中有**:
```yaml

  - "css/custom.css"
```

## 🔍 常见问题排查

### 评论不显示
1. 检查 `enabled: true`
2. 检查 repoId 和 categoryId 是否正确
3. 检查仓库是否公开
4. 检查是否启用了 Discussions

### CSS 不生效
1. 检查文件路径 `assets/css/custom.css`
2. 检查 params.yaml 中的 customCSS 配置
3. 清空浏览器缓存
4. 重启 Hugo 服务器

### 图片不显示
1. 检查图片路径是否正确
2. 检查文件名大小写
3. 使用相对路径或 Page Bundle
