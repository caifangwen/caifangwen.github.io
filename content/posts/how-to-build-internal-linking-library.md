## 前言：为什么说内链库是被严重低估的 SEO 武器

大多数博主谈到 SEO，首先想到的是外链建设（Backlink Acquisition）——去其他网站换链、发 Guest Post、上资源导航。这没有错，但这条路成本高、周期长，而且完全依赖外部资源。

与此同时，一项同样强大却几乎完全掌握在自己手中的技术，却被大多数人忽略了：**内链库（Internal Linking Library）**。

内链就是你博客内部页面之间相互连接的超链接。搜索引擎爬虫（Googlebot）进入你的网站后，会顺着链接抓取所有页面；PageRank 的传递机制意味着一个高权重页面可以通过内链把"权重流量"分配给它所连接的其他页面。换句话说，**一个精心设计的内链网络，可以让你用已有的内容资产为新内容加速排名**。

本文不讲理论，讲的是一套完整的可执行体系：从逻辑架构的设计，到落地执行的具体方法，再到用工具实现自动化管理。如果你是 WordPress 用户、Hugo 用户或者任何自建站博主，这套框架都可以直接复用。

## 第一章：理解内链的权重传递机制

### 1.1 PageRank 与内链的关系

Google 的核心排名算法 PageRank（虽然现在已经是众多因素之一，但其底层逻辑依然有效）的核心思路是：**一个页面被越多高权重页面链接，它自身的权重就越高**。

这一机制同样适用于站内链接。当你的首页（通常权重最高）通过内链指向某篇文章，那篇文章就获得了来自首页的权重传递。这就是为什么把重要文章放在首页导航，或者在首页 Featured 区域做展示，会显著提升那些文章的排名能力。

### 1.2 "权重泄漏"问题

内链管理不当会导致"权重泄漏"：网站的整体权重被稀释到大量低价值页面（比如标签页、归档页、作者页），而真正需要排名的内容页却得不到权重集中。

一个健康的内链体系需要做到：

- **权重向核心页面集中**，而不是平均分散。
- **核心页面的权重能向下传递**给相关内容页。
- **避免孤岛页面**（Orphan Pages）——没有任何内链指向的页面，搜索引擎很难发现和评估。

### 1.3 爬虫行为与"内链深度"

搜索引擎爬虫的抓取预算（Crawl Budget）是有限的。如果你的网站结构混乱，爬虫可能在浅层页面耗尽预算，导致深层的优质内容长期未被抓取。

**黄金法则：网站内任何一个页面，从首页出发，点击不超过 3 次就应该能到达。**

这不是建议，是硬性结构要求。对于一个有数百篇文章的博客来说，实现这一点的唯一方式就是通过精心设计的内链网络——而这正是内链库的核心价值之一。

## 第二章：内链库的逻辑架构——支柱-集群模型

### 2.1 什么是支柱-集群模型（Pillar-Cluster Model）

支柱-集群模型（也叫主题集群，Topic Cluster）是目前内容 SEO 领域最成熟的架构范式。其核心结构如下：

```
支柱页面（Pillar Page）
  ├── 集群页面 A（Cluster Page）
  ├── 集群页面 B（Cluster Page）
  ├── 集群页面 C（Cluster Page）
  └── 集群页面 D（Cluster Page）
```

**支柱页面**：围绕一个核心关键词写的权威性长篇内容，通常是"终极指南"或"完整手册"类型的文章。它覆盖某个主题的所有重要方面，但不会过于深入每一个细节。字数通常在 3000-8000 字之间。

**集群页面**：围绕支柱主题的某一具体方面，进行深度展开的文章。它覆盖一个更窄的关键词，但覆盖得更深。所有集群页面都必须双向连接到对应的支柱页面。

### 2.2 内链连接规则

一套清晰的连接规则是内链库的基础：

**规则一：所有集群页面必须包含至少一个指向支柱页面的内链。**

这是最重要的规则，确保权重能向核心页面汇聚。锚文本应该包含支柱页面的核心关键词。

**规则二：支柱页面必须链接到所有相关的集群页面。**

支柱页面是整个内链网络的枢纽，它需要为爬虫提供一个完整的"导航地图"。

**规则三：相关集群页面之间可以互相连接，但要控制密度。**

同一主题群内的集群页面，如果有天然的引用关系，可以互相连接。但不要为了凑内链而强行连接不相关的内容，这会产生负面效果。

**规则四：跨主题集群的连接要谨慎，需要语义相关性做支撑。**

比如"面料工艺"集群和"服装版型"集群，如果某篇文章确实同时涉及两者，才应该跨集群连接。

### 2.3 建立你的"核心页面清单"

在开始写任何文章之前，你需要有一份文档，列出你的内链架构。这份文档不需要复杂，一个 Excel 表格或 Notion 数据库就足够。

推荐的字段结构：

| 字段 | 说明 |
|---|---|
| 页面标题 | 文章的完整标题 |
| URL/Slug | 页面的永久链接 |
| 核心关键词 | 这个页面主打的关键词 |
| 页面层级 | 支柱页面 / 集群页面 |
| 所属主题群 | 属于哪个支柱页面的集群 |
| 应被哪些页面链接 | 哪些已发布的文章需要加上指向此页的链接 |
| 应链接到哪些页面 | 此页面需要包含哪些内链 |
| 内链完成状态 | 已完成 / 待处理 / 进行中 |

**这份文档就是你的"内链库"核心资产。** 每次发布新文章，更新这张表；每次旧文改版，查阅这张表。

## 第三章：内链的五种落地形式

内链不只是在段落中插入一个超链接那么简单。从用户体验和 SEO 效果两个维度来看，不同形式的内链有着截然不同的效果。

### 3.1 正文内链（Contextual Links）——权重最强

这是 SEO 效果最强的内链形式。内嵌在文章正文段落中，使用包含关键词的锚文本（Anchor Text）指向相关页面。

**示例写法：**

> 在选择面料时，经纬密度是决定手感的核心参数。如果你还没有系统了解过这个指标，可以先阅读我们的[面料经纬密度完整指南](#)，再回来看本文会事半功倍。

这种内链的优势是：
- 上下文相关性强，Google 能理解链接的语义关系。
- 锚文本直接包含目标页面的关键词，权重传递效率最高。
- 用户有明确的点击动机，CTR 高。

**注意事项：** 同一篇文章中，指向同一目标页面的正文内链，只需要出现一次。重复添加相同的内链被 Google 视为过度优化，只计算第一次出现的权重。

### 3.2 推荐阅读模块（Related Posts Block）——点击率最高

在文章中段或末尾，插入一个视觉化的"相关文章"区块。相比纯文字链接，图片+标题+摘要的卡片形式点击率要高出数倍。

**HTML 结构示例（适用于支持自定义组件的博客）：**

```html
<div class="related-posts-grid">
  <h3>延伸阅读</h3>
  <div class="card-grid">
    <a class="post-card" href="/posts/fabric-weave-guide/">
      <img src="/images/fabric-weave-cover.jpg" alt="面料编织结构指南">
      <div class="card-content">
        <h4>面料编织结构完全指南：平纹、斜纹、缎纹的区别与应用</h4>
        <p>深入解析三大基础编织结构的形成原理、外观特性与适用场景。</p>
      </div>
    </a>
    <!-- 更多卡片 -->
  </div>
</div>
```

**Hugo Shortcode 版本：**

```
{{< related-posts >}}
  {{< post-card href="/posts/fabric-weave-guide/" title="面料编织结构完全指南" >}}
  {{< post-card href="/posts/thread-count-explained/" title="经纬密度详解：数字越大越好吗？" >}}
  {{< post-card href="/posts/fabric-care-labels/" title="洗涤标签完全解读" >}}
{{< /related-posts >}}
```

### 3.3 系列导航条（Series Navigation Bar）——停留时长最佳

当你的内容是系列教程时，在文章顶部放一个醒目的导航提示条，告诉读者这是系列的第几部分，并提供整个系列的链接。

**这一设计的 SEO 价值体现在两个维度：**

1. **内链价值**：每篇系列文章都指向聚合页（系列首页），聚合页同时链接回所有系列文章，形成强健的内链网络。
2. **用户行为价值**：用户读完当前篇会自然跳转到下一篇，显著增加每次会话的页面浏览量和站内停留时间，这两个指标都是 Google 排名的正向信号。

**Markdown/Hugo 写法（使用 Callout 组件）：**

```markdown
{{< callout type="info" >}}
📚 **本文是《面料工艺从入门到精通》系列的第 3 篇。**  
[← 上一篇：纤维分类与特性](/posts/fiber-types/) ｜ [查看完整系列目录](/series/fabric-guide/) ｜ [下一篇：染色工艺原理 →](/posts/dyeing-process/)
{{< /callout >}}
```

### 3.4 步骤关联内链（Process Step Links）——深度阅读场景

在教程类文章中，如果某个步骤需要较多背景知识，可以在步骤标题或步骤说明中直接嵌入内链，引导用户去读对应的前置内容，读完后再回来继续。

**示例：**

```html
<div class="process-step">
  <span class="step-number">步骤 2</span>
  <h3>选择合适的面料克重</h3>
  <p>克重（GSM，Grams per Square Meter）直接影响成衣的垂感和保暖性。
     如果你对 GSM 不熟悉，建议先阅读
     <a href="/posts/gsm-guide/">面料克重完整指南</a>，
     再回来完成这一步。</p>
</div>
```

### 3.5 脚注式内链（Footnote Links）——学术内容场景

对于数据引用、术语解释或延伸阅读，可以使用类似学术论文的脚注形式添加内链。这种形式在用户体验上干扰性最低，适合深度技术类内容。

在 Markdown 中：

```markdown
面料的耐磨性通常用马丁代尔磨耗次数[^1]来衡量，一般家用面料要求超过 15,000 次，商用场景则需要 30,000 次以上。

[^1]: [马丁代尔磨耗测试方法详解](/posts/martindale-test-guide/)
```

## 第四章：锚文本策略——内链的"命名艺术"

很多人只注意到内链要"加链接"，却忽视了锚文本（Anchor Text）的选择同样至关重要。锚文本是链接的可见文字，它告诉搜索引擎这个被链接的页面"是关于什么的"。

### 4.1 锚文本的五种类型

**精确匹配锚文本（Exact Match）**：锚文本就是目标页面的核心关键词。
- 示例：`<a href="/posts/gsm-guide/">面料克重 GSM</a>`
- 优点：权重传递精准。
- 风险：过度使用会触发 Google 的过度优化惩罚。

**局部匹配锚文本（Partial Match）**：包含目标关键词但加了修饰语。
- 示例：`<a href="/posts/gsm-guide/">了解面料克重的完整指南</a>`
- 这是最推荐的日常写法，自然且有效。

**品牌锚文本（Branded）**：使用你的品牌名或网站名作为锚文本。适合指向首页或品牌主页。

**通用锚文本（Generic）**：如"点击这里"、"查看更多"、"了解详情"。
- 这类锚文本对 SEO 价值最低，尽量避免。唯一可接受的场景是用在按钮上，按钮本身已有视觉上下文。

**裸 URL 锚文本（Naked URL）**：直接显示 URL 地址。不推荐在正文中使用，视觉上突兀且对 SEO 无额外价值。

### 4.2 锚文本多样化原则

一个健康的内链库中，指向同一目标页面的所有内链，其锚文本应该是多样化的，而不是千篇一律地使用同一个词组。

**示例：同一篇支柱文章可以被这样链接：**

- `面料克重完整指南`
- `了解 GSM 的详细解释`
- `这篇关于克重的深度分析`
- `面料 GSM 参数详解`
- `克重对垂感的影响`

这种多样性让链接看起来更自然，同时也让目标页面能匹配更多语义相关的长尾查询。

### 4.3 内链锚文本的"禁忌"

1. **不要用完全相同的锚文本反复链接同一个页面**：这是明显的人工干预信号。
2. **不要在一篇文章中放 5 个以上指向同一页面的内链**：通常 1-2 个就够了。
3. **不要把无关内容强行用关键词锚文本连接**：相关性缺失会让 Google 困惑，甚至认为是操纵行为。
4. **不要用纯图片做内链而不加 Alt 属性**：如果内链是一张图片，确保图片有描述性的 alt 文本，这相当于图片内链的锚文本。

## 第五章：工具层——如何自动化管理内链库

手动管理几十篇文章的内链勉强可行，但当你的博客规模扩大到数百篇文章时，手动维护的成本会急剧上升。这时候需要工具来帮助自动化。

### 5.1 WordPress 用户的工具选择

**Link Whisper（付费，约 $77/年）**

这是目前 WordPress SEO 圈公认最成熟的内链自动化工具。核心功能包括：

- **智能建议**：在你写文章时，实时分析文章内容，建议你应该插入哪些内链，以及链向哪些已有页面。
- **批量添加**：可以为某个关键词设置规则，在全站范围内批量插入内链（这一功能要谨慎使用，建议限制每篇文章的自动添加数量）。
- **孤岛页面报告**：自动找出没有任何内链指向的页面，方便你逐一补充。
- **内链数据仪表盘**：显示每个页面有多少内链指向它、从它出发有多少内链，便于识别权重不均衡的情况。

**Internal Link Juicer（免费版可用）**

更轻量的选择。主要功能是关键词自动映射：你在后台设置规则，比如凡是文章中出现"面料工艺"，就自动把这个词变成指向指定页面的内链。

适合预算有限或内链需求相对简单的博主。

**Yoast SEO（免费/付费）**

Yoast 内置了基础的"内链建议"功能，在编辑文章时会在侧栏推荐相关文章。虽然不如 Link Whisper 强大，但对于轻度用户已经足够。

### 5.2 Hugo / 静态站点用户的工具方案

静态站点无法使用 WordPress 插件，但有同样有效的解决方案。

**方案一：利用 Hugo 的 Related Pages 功能**

Hugo 内置了基于标签、分类和自定义分类法的相关文章推荐系统。在 `config.toml` 中配置：

```toml
[related]
  includeNewer = true
  threshold = 80
  toLower = false

  [[related.indices]]
    name = "keywords"
    weight = 100

  [[related.indices]]
    name = "tags"
    weight = 80

  [[related.indices]]
    name = "categories"
    weight = 60
```

然后在文章模板（`layouts/_default/single.html`）中调用：

```html
{{ $related := .Site.RegularPages.Related . | first 4 }}
{{ with $related }}
<aside class="related-posts">
  <h3>相关文章</h3>
  <div class="related-grid">
    {{ range . }}
    <a class="related-card" href="{{ .Permalink }}">
      {{ with .Params.cover }}
      <img src="{{ .image }}" alt="{{ .alt }}">
      {{ end }}
      <div>
        <h4>{{ .Title }}</h4>
        <p>{{ .Description }}</p>
      </div>
    </a>
    {{ end }}
  </div>
</aside>
{{ end }}
```

这样每篇文章底部会自动渲染出基于内容相关性的推荐模块，所有推荐链接都是内链。

**方案二：Shortcode 化的内链组件**

在 Hugo 中创建一个 Shortcode（`layouts/shortcodes/internal-link.html`）：

```html
{{- $href := .Get "href" -}}
{{- $title := .Get "title" -}}
{{- $desc := .Get "desc" | default "" -}}
<a class="internal-link-card" href="{{ $href }}">
  <strong>{{ $title }}</strong>
  {{ with $desc }}<span>{{ . }}</span>{{ end }}
</a>
```

在文章中使用：

```markdown
{{< internal-link href="/posts/gsm-guide/" title="面料克重 GSM 完整指南" desc="深度解析克重对面料性能的影响" >}}
```

**方案三：维护一份 JSON 内链字典**

在 Hugo 项目的 `data/` 目录下创建 `internal-links.json`：

```json
{
  "gsm-guide": {
    "title": "面料克重 GSM 完整指南",
    "url": "/posts/gsm-guide/",
    "keywords": ["克重", "GSM", "面料厚度"]
  },
  "fabric-weave": {
    "title": "面料编织结构完全指南",
    "url": "/posts/fabric-weave-guide/",
    "keywords": ["编织结构", "平纹", "斜纹", "缎纹"]
  }
}
```

这份字典就是你的内链数据库。当 URL 需要更新时，只需要修改这一个文件，全站所有引用这个 key 的 Shortcode 都会自动更新——这与 WordPress 的 functions.php 短代码字典方案在逻辑上是等价的。

### 5.3 通用方案：Search & Replace 工作流

无论你用什么平台，以下工作流都适用：

1. **写文章时**：在需要放内链的地方，先放一个占位符，比如 `[[LINK:GSM_GUIDE]]`。
2. **发布前**：用编辑器的全局搜索替换功能（或脚本），把所有占位符替换成真实的 HTML/Markdown 链接。
3. **URL 变更时**：只需要更新你的占位符字典，然后全局替换一次。

WordPress 上，可以用 **Search Regex** 插件执行这个替换操作；Hugo 项目中，可以用 `sed` 或 Python 脚本批量处理 Markdown 文件。

## 第六章：内链库的建设时序——从第一篇文章开始

### 6.1 "先鸡后蛋"的启动问题

很多人问：内链库需要已有文章才能建，但文章里需要内链……这是个先有鸡还是先有蛋的问题。

解决方案是：**先用占位符建立内链骨架，后期替换为真实链接。**

具体操作步骤：

**第一步：在写第一批文章之前，先规划好核心页面清单。**

哪怕你还没写这些文章，也先把它们的 Slug 定好，例如：

- `/posts/fabric-ultimate-guide/`（支柱页面）
- `/posts/gsm-guide/`（集群页面）
- `/posts/weave-structure/`（集群页面）
- `/posts/fabric-care/`（集群页面）

**第二步：每篇新文章在生成时，就在末尾加入带有占位链接的推荐阅读模块。**

```html
<div class="related-posts">
  <h3>延伸阅读</h3>
  <ul>
    <li><a href="/posts/fabric-ultimate-guide/">面料工艺终极指南（即将发布）</a></li>
    <li><a href="/posts/gsm-guide/">面料克重 GSM 完整指南（即将发布）</a></li>
  </ul>
</div>
```

即使目标页面还没有发布，这些链接就已经建立在 HTML 结构中了。当目标页面发布后，链接立刻生效，不需要再回来修改任何文章。

**第三步：目标页面发布时，更新内链库文档，标记相关链接为"已激活"。**

### 6.2 批量补链工作流

对于已有大量文章但内链混乱的博主，推荐的补链工作流：

1. **用 Screaming Frog（免费版支持 500 URL）爬取全站**，导出所有页面的内链数据。
2. **在 Excel 中对照你的核心页面清单**，找出哪些核心页面的入链数量明显低于其他页面。
3. **从最高权重的已发布文章开始**，逐一为入链不足的核心页面补充正文内链。
4. **每次补链操作后，更新内链库文档。**

不要试图一次性把所有内链都做完。按重要性排序，每周更新 5-10 篇文章的内链，持续 4-6 周，就能让你的内链网络大幅改善。

### 6.3 内链库的日常维护节奏

建立内链库不是一次性工作，而是一个持续的运营动作。推荐的维护节奏：

**每次发布新文章时：**
- 检查内链库文档，找出与新文章相关的已有核心页面，在新文章中添加指向它们的内链。
- 在内链库文档中更新：哪些旧文章需要补充指向这篇新文章的内链（列入待处理队列）。

**每月一次：**
- 检查"待处理内链"队列，批量处理上个月积累的补链任务。
- 用 Screaming Frog 或 Google Search Console 检查是否有新增的孤岛页面。
- 检查是否有页面的 URL 发生了变化，及时更新内链库字典。

**每季度一次：**
- 审查核心页面的入链数量，确保每个支柱页面有足够的内链支撑。
- 根据排名数据，调整内链库的重点——对排名上升明显的页面加大内链投入。

## 第七章：常见错误与防扣分指南

### 7.1 错误一：过度内链

一篇 2000 字的文章放了 30 个内链，每隔一句话就有一个超链接——这不是"内链丰富"，这是"链接农场"的特征之一，会让 Google 认为你在人工操纵链接结构。

**经验参考：** 每 300-500 字的文章，放 1-2 个正文内链是合理的。加上文末的推荐阅读模块（通常 3-4 个），一篇 2000 字的文章总内链数控制在 8-12 个以内是安全的。

### 7.2 错误二：无关强连

把"面料工艺"的文章硬性连到"旅游攻略"，只是为了增加某个页面的入链数量。

**Google 的语义理解已经相当成熟。** 无关的链接不会传递正向权重，反而会传递"主题混乱"的信号，影响整个网站的主题权威性（Topical Authority）。

**所有内链都应该基于真实的语义相关性，而不是单纯的权重传递需求。**

### 7.3 错误三：链接孤岛

发布了一篇文章，但没有任何其他文章链向它。这篇文章就成了"孤岛页面"（Orphan Page）。

孤岛页面存在两个问题：爬虫难以发现它（或者抓取频率极低），以及它无法从其他页面获得任何权重传递，排名能力极弱。

**每次发布新文章，至少要确保有 2-3 篇已有文章通过内链指向它。**

### 7.4 错误四：内链死链

目标页面的 URL 发生了变化，但旧的内链指向了已经 404 的地址。这不仅浪费爬虫资源，还会损害用户体验。

**预防措施：**
- 一旦 Slug 确定，尽量不要修改。
- 如果必须修改 URL，一定要在服务器配置 301 重定向，同时更新内链库字典并批量替换文中的链接。
- 定期用 Screaming Frog 或 Ahrefs 的站内检测功能扫描死链。

### 7.5 错误五：重复锚文本

全站所有指向支柱页面的内链，锚文本都是一模一样的精确匹配关键词。

虽然这不会直接导致惩罚，但这种高度规律性是人工干预的明显特征。更重要的是，多样化的锚文本能帮助目标页面匹配更多语义变体的搜索查询。

## 第八章：用 AI 批量生成带内链骨架的文章

既然内链体系需要在文章发布时就已经建立好，那么最高效的做法是：**让 AI 在生成文章时就自动嵌入内链结构**。

### 8.1 给 AI 的内链生成指令模板

当你用 AI 工具（ChatGPT、Claude 等）生成博客文章时，在提示词中加入以下内链相关指令：

```
请在文章的以下位置加入内链结构：

1. 在文章正文中，每出现以下关键词时，在第一次出现处加上 HTML 超链接标记（用 [[PLACEHOLDER]] 格式，我后期会替换真实 URL）：
   - "面料克重" → [[LINK:GSM_GUIDE]]
   - "编织结构" → [[LINK:WEAVE_GUIDE]]
   - "洗涤护理" → [[LINK:CARE_GUIDE]]

2. 在文章末尾，加入一个 class 为 "related-posts-grid" 的 div，包含 3 个指向以下占位 URL 的卡片：
   - /posts/fabric-ultimate-guide/
   - /posts/gsm-guide/
   - /posts/weave-structure/
   每个卡片包含 title 和简短 description 字段。

3. 在文章开头加入一个 Callout，说明本文属于 [面料工艺系列] 的一部分，并包含指向 /series/fabric-guide/ 的链接。
```

### 8.2 批量处理的 Hugo 工作流

对于 Hugo 用户，可以用以下 Shell 脚本批量处理占位符替换：

```bash
#!/bin/bash
# replace-internal-links.sh
# 将占位符替换为真实的 Hugo 内链格式

find ./content -name "*.md" -type f | while read file; do
  # 替换 GSM 指南占位符
  sed -i 's|\[\[LINK:GSM_GUIDE\]\]|[面料克重 GSM 完整指南](/posts/gsm-guide/)|g' "$file"
  
  # 替换编织结构指南占位符
  sed -i 's|\[\[LINK:WEAVE_GUIDE\]\]|[面料编织结构指南](/posts/weave-structure/)|g' "$file"
  
  # 替换洗涤护理指南占位符
  sed -i 's|\[\[LINK:CARE_GUIDE\]\]|[面料洗涤护理完整指南](/posts/fabric-care/)|g' "$file"
  
  echo "Processed: $file"
done
```

**这个工作流的核心优势：** 你现在写的每一篇文章，都自带完整的内链骨架。当目标页面发布后，运行一次脚本，全站内链立即激活。URL 需要变更时，只需要修改脚本中的目标 URL，重新运行一次，全站同步更新。

## 结语：把内链库变成你的复利资产

内链库最大的特性是**复利效应**：

- 你今天建立的内链网络，会在未来每一篇新文章发布时发挥作用。
- 你今天确定的核心页面清单，是未来所有内容创作的导航地图。
- 你今天在 AI 指令中加入的内链骨架要求，会让每一篇生成的文章都自动融入你的内链体系。

这与外链建设的最大区别是：**外链需要持续地向外部"索取"，而内链是完全掌握在自己手中的主动资产。**

一个有 200 篇文章、内链结构完整的博客，在权重积累速度上，可以超越一个有 500 篇文章但内链混乱的博客。这不是夸张，这是搜索引擎评估"主题权威性"（Topical Authority）的基本逻辑。

**现在就可以开始的三个行动：**

1. 用 30 分钟，写下你最核心的 5 个支柱页面的标题和 Slug——哪怕它们还没有写。
2. 给你的 AI 写作工具加上一条内链生成指令，从下一篇文章开始执行。
3. 创建一个 Notion 或 Excel 文档，开始记录你的内链库清单。

内链库不需要等到网站"做大"了再建。**越早建立，复利越大。**
