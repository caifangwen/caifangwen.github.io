## 什么是支柱页（Pillar Page）

支柱页是覆盖某一**核心主题**的长篇综合性页面，它不深挖每个细节，而是给出全局框架，并通过内部链接指向各篇深度子文章（Cluster Content）。

三者关系如下：

```
支柱页（Pillar Page）
  ├── 子文章 A（聚焦长尾关键词）
  ├── 子文章 B
  ├── 子文章 C
  └── 子文章 D ...
```

Google 通过这种**主题集群（Topic Cluster）**结构判断你的站点在某领域是否具备权威性，从而提升整组页面的排名。

## 你已经有很多文章了——为什么现在更需要支柱页

| 常见问题 | 支柱页的作用 |
|---|---|
| 文章孤岛，互相不链接 | 作为枢纽，统一向外辐射链接 |
| 多篇文章抢同一关键词 | 支柱页承接主词，子文章承接长尾词 |
| 新文章权重积累慢 | 支柱页的权重通过内链传递给子文章 |
| 用户读完一篇就离开 | 引导用户在集群内持续浏览 |

## WordPress 支柱页用什么形式最好

### 推荐形式：**长篇"终极指南"式页面（Ultimate Guide）**

这是最常见、SEO 效果最好的形式，特点：

- 字数通常 **3000–8000 字**
- 覆盖主题的所有一级子问题，每个子问题写 200–400 字概述
- 每个子问题末尾放**跳转链接**，指向对应深度子文章
- 页面顶部放**锚点目录（TOC）**，提升可读性与停留时间

### 其他可选形式

| 形式 | 适合场景 | WordPress 实现 |
|---|---|---|
| 终极指南（推荐） | 博客、知识站、工具站 | 普通 Page 或 Post |
| 资源汇总页 | 工具/教程类站点 | Page + 自定义模板 |
| FAQ 集合页 | 问答型内容 | Page + FAQ Schema |
| 视觉化 Hub 页 | 品牌站、产品站 | Page Builder（Elementor 等） |

**结论：大多数 WordPress 站点，用"普通 Page + 终极指南结构"即可，不需要额外插件。**

## WordPress 支柱页实操步骤

### 第一步：整理现有文章，划分主题集群

把你已有的文章列出来，按主题分组：

```
主题：WordPress SEO
  - 如何做关键词研究（已有）
  - 内链建设技巧（已有）
  - 图片 SEO 优化（已有）
  - XML Sitemap 设置（已有）
  ↑ 这组文章 → 需要一篇"WordPress SEO 完整指南"作为支柱页
```

### 第二步：创建支柱页

1. 后台 → **页面 → 新建页面**（用 Page，而非 Post，便于管理且不受分类影响）
2. 设置一个**宽泛的核心关键词**作为页面主题（如"WordPress SEO"）
3. 结构建议：
   ```
   H1：[主题] 完整指南（2026 最新）
   引言：为什么这个主题重要
   H2：子话题一（概述 + 内链到子文章）
   H2：子话题二
   ...
   H2：常见问题 FAQ
   结尾：总结 + CTA
   ```

### 第三步：建立双向内链

- **支柱页 → 子文章**：在每个 H2 区块末尾加"深入阅读"链接
- **子文章 → 支柱页**：在每篇子文章的引言或结尾，加一句"本文是《XX 完整指南》的一部分，[点击查看完整指南]"

> ⚠️ 双向链接是关键，很多人只做单向，效果大打折扣。

### 第四步：SEO 设置（以 Rank Math / Yoast 为例）

| 设置项 | 建议值 |
|---|---|
| Focus Keyword | 核心宽泛词（如"WordPress SEO"） |
| Slug | `wordpress-seo-guide`（简洁，含核心词） |
| Meta Description | 强调"完整指南"、"X 个步骤"等 |
| Schema Type | Article 或 HowTo |
| Canonical | 指向自身，避免重复 |

### 第五步：更新旧文章

把已有子文章逐一打开，在合适位置加入指向支柱页的内链。这一步不要跳过，是整个集群生效的关键。

## 支柱页的 URL 结构建议

```
# 好的示例
yourdomain.com/wordpress-seo-guide/
yourdomain.com/content-marketing/

# 避免
yourdomain.com/?p=1234
yourdomain.com/2026/04/12/wordpress-seo-guide/  ← 含日期，支柱页不适合
```

在 WordPress 设置 → 固定链接中，选择"文章名"模式，然后手动设置每篇页面的 Slug。

## 常见误区

1. **把支柱页写成普通博客文章**：支柱页应该是"地图"，不是"目的地"，不要把所有细节都塞进去。
2. **只建了支柱页，没有子文章支撑**：集群内容越丰富，支柱页排名越稳。你已有很多文章，这正是你的优势。
3. **支柱页放在 Post 里加了发布日期**：建议用 Page 类型，或在 Post 中隐藏日期，避免给用户"过时"的感觉。
4. **内链锚文本全用"点击这里"**：锚文本应包含目标关键词，如"查看 WordPress 图片优化详细教程"。

## 总结

支柱页不需要从零开始写内容——你已有的大量文章就是最好的原材料。现在要做的是：**选一个核心主题，写一篇"地图式"综合页，把散落的文章用内链串起来**，让 Google 和用户都能看清你的内容体系。

形式上，WordPress 用**普通 Page + 终极指南结构**是最稳妥、维护成本最低的选择。
