## 一、为什么要用 AI 批量处理 TDK？

利用 AI 批量处理 TDK（Title 标题、Description 描述、Keywords 关键词）是解决独立站与国际站"内容重复"问题最高效的手段。目前主要有两种实现路径：**插件派**（简单快速）和**表格 + API 派**（高度可控）。

---

## 二、AI 批量生成 TDK 的两种路径

### 方案一：WordPress 插件（最快，适合非技术人员）

现在有很多 WooCommerce 插件直接集成了 OpenAI（GPT）或 Claude 的 API，可以在后台批量生成 TDK。

**推荐插件：**

- **AI Product Tools / AI Power：** 选中多个产品，输入一个 Prompt，比如"请根据产品标题和属性，生成一段 150 字符以内、包含 [Brand Name] 的 SEO 描述"，一键批量更新。
- **Rank Math / Yoast SEO AI Add-on：** 这两款主流 SEO 插件均已内置 AI 辅助功能，可批量生成 Meta Title 和 Description。

| | 插件方案 |
|---|---|
| 优点 | 无需接触代码，直接在 Woodmart 后台操作，预览方便 |
| 缺点 | 灵活度略低，部分高级功能需付费订阅 |

---

### 方案二：导出 CSV + Python / Google Sheets（最可控，适合大批量）

若有几千个产品且想节省插件费用，直接用 API 处理表格是最优选。

**具体流程：**

**第一步：导出**

使用 **WP All Export** 插件，导出产品的 `ID`、`Title`、`Original Description`、`Attributes`（参数）到 Excel 或 CSV。

**第二步：AI 处理（二选一）**

- **Google Sheets + 脚本：** 将数据上传到 Google Sheets，安装支持 GPT API 的扩展（如 *GPT for Sheets and Docs*），在描述列旁边写公式：
  ```
  =GPT("请为这个产品写一个SEO标题", A2)
  ```
- **Python 脚本：** 读取 CSV，调用 OpenAI 的 Chat Completion 接口，将生成的新 TDK 存入新列。

**第三步：导入**

使用 **WP All Import** 插件将处理好的 CSV 重新导入，在导入设置里将 AI 生成的列分别映射到对应字段：

```
AI生成标题列  →  _rank_math_title / _yoast_wpseo_title
AI生成描述列  →  _rank_math_description / _yoast_wpseo_metadesc
```

## 三、Prompt 策略（核心）

为了让 AI 生成的内容避免被谷歌判定为重复，Prompt 应包含以下逻辑：

```
你是一个资深的跨境电商 SEO 专家。我会给你一个产品的原始参数和标题（来自阿里巴巴）。
请为我生成：

1. 一个吸引人的 SEO 标题（60字符内），必须包含品牌名 [Your Brand]
2. 一段专业的 Meta Description（160字符内），强调"Direct Factory Price"和"Global Shipping"
3. 5个核心 SEO 标签（Tags）

注意：语言风格偏向 B2B 采购商，避免过度夸张，确保内容与阿里原件有至少 40% 的文本差异。
```

**为什么这套 Prompt 有效：**

- **瞬间解决收录问题：** 几百个产品一次性导入后，TDK 全是 AI 重新原创的，谷歌会认为这是一个全新的高质量品牌站，而不是阿里的镜像。
- **保持 Woodmart 样式整洁：** AI 可以统一标题长度，将所有标题改成`产品名 + 核心规格 + 品牌名`的格式，在 Woodmart 网格视图（Grid View）下高度一致，非常美观。

## 四、快速拉起独立站：两阶段策略

对于独立站来说，**全站空白是最糟糕的状态**（影响搜索引擎收录和客户信任）。需要先让网站"像个样子"，再逐步优化。

### 第一阶段：爆破式导入（The Skeleton）

**目的：** 快速填充内容，建立网站架构，让谷歌爬虫开始工作。

**操作步骤：**

1. 使用 **WP All Import** 插件，将公司国际站所有产品基础数据（标题、价格、主图、基础参数）一次性批量导入
2. 导入时统一将产品划归到正确的 **Categories（分类）**
3. 状态设为 **Published（已发布）**，并给详情页挂载全局公司介绍模块

**好处：** 网站瞬间拥有几百到上千个页面，看起来是成熟的供应商，分类清晰，用户可以搜索到。

> **关键提醒：** 超过 1000 个产品导入时，注意以下两点：
> - **图片缩略图生成** 会短时间内消耗大量 CPU，可在服务器低峰期操作
> - **导入完成后清理缓存**（WP Rocket 或 Woodmart 自带的 CSS/JS 缓存），确保新产品样式正常显示

---

### 第二阶段：节奏化精修（The Pulse）

**目的：** 解决 SEO 重复问题，提高核心产品转化率。

**操作步骤：**

1. 从所有产品中选出 **20% 的核心利润产品（爆款）**
2. 使用 **Woodmart Layout Builder** 为核心产品设计独特页面布局
3. 配合 **ACF** 填充更详细的参数，并重写这 20% 产品的标题和描述
4. 剩下 80% 的非核心产品保持原样，作为网站的**长尾流量**支撑

**好处：** 集中精力办大事。只要 20% 的核心产品排在谷歌前面，生意就活了。

## 五、快速拉起独立站的 4 个"加速器"

### 加速器 1：使用 Woodmart Demo 内容

不要从 0 开始设计页面。直接在 `Woodmart → Dummy Content` 里导入一个与行业接近的 Demo。

- 瞬间配置好导航栏、页脚、颜色风格
- 删掉演示产品，换成批量导入的真实产品即可

### 加速器 2：全局 HTML Block 挂载

在还没来得及为每个产品写文案时：

1. 制作一个精美的 HTML Block（包含公司实力介绍、工厂视频、认证证书、联系方式）
2. 在 `Theme Settings → Single Product → After content` 里将其挂载到**所有产品页面底部**

效果：即使产品描述只有几行字，整个页面依然饱满、专业。

### 加速器 3：AI 批量改写描述

针对几百个产品描述的 SEO 改写，使用以下流程：

```
导出产品 CSV → 丢给 ChatGPT / Claude 批量改写 → 重新导入 WooCommerce
```

Prompt 示例：
```
请根据以下产品参数，为我的独立站重写一段品牌风格的描述，
避免堆砌关键词，字数在 200 字左右，语气偏向专业 B2B 采购商。
```

### 加速器 4：开启 AJAX Filter（搜索与过滤）

Woodmart 最强大的功能之一是边栏过滤（AJAX Filters）。

- 批量导入产品并分好类别后，在**分类页开启 AJAX Filters**
- 让客户在几百个产品中快速筛选规格
- 这种"丝滑"的体验是国际站平台给不了的差异化优势

## 六、"一次性导入"与"持续更新"的正确理解

很多人认为两者冲突，其实它们是两个维度的操作。

### 一次性导入没问题的原因

- **建立类目权重：** 产品丰富度本身是专业性的体现。500 个产品让谷歌爬虫意识到这是一个完整的"行业供应库"，有利于关键词覆盖。
- **缩短空窗期：** 一次性填满，让偶尔访问的客户觉得公司实力雄厚，而不是只有寥寥几个样品的"小作坊"。

### "持续更新"的真正含义

持续更新**不是**让你每天手动传一个产品，而是给谷歌提供**鲜活度信号（Freshness Signal）**：

- **爬虫抓取频率：** 如果站点三年不动，Googlebot 来访频率会越来越低；规律更新能让爬虫养成定期抓取的习惯。
- **SEO 微操：** 谷歌喜欢看到网页在进化——今天优化某个产品标题，明天给某个产品加视频。这些细微更新比单纯增加产品数量对排名提升更有帮助。
- **用户留存：** 老客户看到"New Arrivals"或"News"有更新，会觉得这家公司经营状态良好。

---

## 七、避坑提醒

> **千万不要** 一个一个手动上传产品。这会让你陷入无尽的琐碎工作中，导致网站迟迟无法上线。

独立站的核心逻辑是：**先上线，再优化。**

---

## 八、推荐操作节奏

```
第 1 天   把所有产品通过 CSV 一次性灌进去
第 2-3 天 调好 Woodmart 全局样式（Logo、颜色、页脚、全局 HTML Block）
此后每天  优化 3-5 个核心产品（描述、布局、ACF 参数、实拍图）
每 周     发 1 篇 Blog/News 行业文章（AI 生成后人工微调）
```

---

*产品数量较大（几千个）时，建议先用插件或 Python 脚本完成 AI 批量改写，再统一导入，避免反复操作。*
