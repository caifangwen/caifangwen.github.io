# n8n 详细介绍

## 什么是 n8n？

n8n（读作 "n-eight-n"）是一个**开源的工作流自动化平台**，允许用户通过可视化界面连接不同的应用程序、服务和 API，实现业务流程的自动化，无需编写大量代码。

它的名字来源于 "node" 中间有 8 个字母（n-ode-automatio-n），是 "nodemation" 的缩写。

---

## 核心特点

### 🔓 开源 & 可自托管
- 代码完全开源（Fair-code 协议）
- 可以部署在自己的服务器上，数据完全掌控在自己手中
- 也提供云端托管版本（n8n Cloud）

### 🎨 可视化拖拽界面
- 通过节点（Node）连接构建工作流
- 直观的画布式编辑器
- 实时查看数据流转

### 🔗 丰富的集成
- 支持 **400+** 应用和服务的原生集成
- 包括：Slack、Gmail、GitHub、Notion、MySQL、PostgreSQL、Stripe、Salesforce 等
- 支持任意 HTTP/REST API、GraphQL、WebSocket

### 💻 代码能力
- 可以在节点中编写 **JavaScript / Python** 代码
- 支持自定义逻辑，不受限于预设功能

---

## 核心概念

| 概念 | 说明 |
|------|------|
| **Workflow（工作流）** | 自动化任务的完整流程图 |
| **Node（节点）** | 工作流中的每个步骤/动作单元 |
| **Trigger（触发器）** | 启动工作流的事件（定时、Webhook、手动等） |
| **Credential（凭证）** | 安全存储的 API Key、账号密码等 |
| **Expression（表达式）** | 动态引用上游节点数据的语法 |

## 典型使用场景

### 📧 数据同步 & 通知
- 新用户注册 → 自动发送欢迎邮件 + 添加到 CRM
- 监控网站状态 → 故障时发送 Slack/钉钉告警

### 📊 数据处理
- 定时抓取数据 → 清洗转换 → 写入数据库/Excel
- 多数据源聚合报表，自动发送给团队

### 🤖 AI 工作流（近年重点方向）
- 集成 OpenAI / Claude / Ollama
- 构建 AI Agent，自动处理邮件、客服问题
- RAG（检索增强生成）流程搭建

### 🛒 电商 & 业务自动化
- 订单处理 → 物流通知 → 库存更新
- 社交媒体内容自动发布

### 🔄 DevOps & IT 运维
- GitHub PR 合并 → 触发部署流程
- 日志监控 → 自动创建 Jira 工单

## 部署方式

```bash
# Docker 快速启动（最常用）
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v ~/.n8n:/home/node/.n8n \
  docker.n8n.io/n8nio/n8n

# 访问 http://localhost:5678
```

也支持：
- **npm** 安装
- **Docker Compose**（生产推荐）
- **Kubernetes** 部署
- **n8n Cloud** 云托管（付费）

## 总结

n8n 最大的优势是**开源 + 自托管 + 强大的代码能力**，非常适合：
- 有一定技术能力的开发者或技术团队
- 对数据隐私有要求的企业
- 需要复杂自定义逻辑的自动化场景
- 构建 AI Agent 工作流

如果你想零代码、开箱即用，Zapier 更友好；如果你想要灵活、可控、省钱，n8n 是绝佳选择。
