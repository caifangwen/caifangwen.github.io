## 一、背景与问题诊断

### 1.1 典型症状

大多数依赖博客引流的 ToB WordPress 电商站，流量结构往往呈现以下特征：

- **博客页**承担 80%~95% 的自然搜索流量
- **产品页**排名差、流量极少，基本只靠直接访问或博客跳转
- 产品页内容只有：产品名称 + 参数表格 + 几张产品图
- 产品页缺少 TDK（Title / Description / Keywords），部分站点甚至用插件默认格式生成，如 `产品名 - 站点名`，无任何关键词优化
- 博客与产品页之间没有系统性内链，流量进来了但无处承接

### 1.2 本质问题拆解

这个状态在 SEO 层面造成两个结构性缺陷：

**缺陷一：漏斗断层**

用户通过博客进入站点，获取了信息，但页面内没有足够的路径引导他们进入产品页，最终跳出。Google Analytics 里表现为博客页跳出率高、会话深度浅、产品页几乎没有从博客导入的流量。

**缺陷二：权重孤岛**

产品页没有内链指入（或极少），又没有优质内容，Google 的 PageRank 传导链在博客层就断掉了。产品页内容单薄，Google 无法判断这个页面"值得排在前面"，因此产品词的自然排名长期停滞在第 3~5 页。

### 1.3 改造目标

本文的核心目标是：

> **保住现有博客排名 → 用内链漏斗承接博客流量 → 丰富产品页内容 → 提升产品页在 Google 的权重 → 产品词排名上升 → 询盘/加购转化提升**

整个改造分六个模块，按优先级排序执行，最高优先级动作可在 1~2 周内完成并见效。

## 二、流量审计：改造前必做的基础工作

在动手之前，必须搞清楚现有流量的结构，否则改造方向会走偏。

### 2.1 Google Search Console 导出博客流量清单

进入 GSC → 效果 → 页面，按点击量从高到低排序，导出前 50 篇博客文章的数据，记录以下字段：

| 字段 | 说明 |
|---|---|
| 页面 URL | 博客文章地址 |
| 点击量（近 3 个月） | 衡量流量权重 |
| 展示量 | 衡量该页面的索引覆盖面 |
| 平均排名 | 判断是否值得重点维护 |
| 核心查询词 | 该页面主要靠什么词排名 |

导出后，人工为每篇文章标注两个维度：

**① 搜索意图分类**

| 类型 | 特征 | 示例查询词 |
|---|---|---|
| 信息型（Informational） | 用户在了解概念、方法、原理 | "温湿度传感器是什么"、"RS485 接口怎么接线" |
| 对比型（Comparative） | 用户在比较选项 | "温湿度传感器哪个品牌好"、"RS485 vs RS232 区别" |
| 商业调研型（Commercial） | 用户有购买意向，在做决策前调研 | "工业温湿度传感器价格"、"食品冷链传感器选型" |
| 交易型（Transactional） | 用户已决定购买，在找具体产品 | "RS485 温湿度传感器购买"、"XYZ-200 传感器报价" |

商业调研型和交易型文章是内链改造的最高优先级，这些用户距离购买最近。

**② 关联产品页映射**

为每篇博客找到它最匹配的 1~2 个产品页，记录在表格里，这是后续内链植入的操作依据。

```
示例映射表：

博客：/blog/how-to-choose-industrial-humidity-sensor
  → 关联产品页：/products/rs485-humidity-sensor-xyz200
  → 搜索意图：商业调研型
  → 月点击量：2,800
  → 优先级：P0

博客：/blog/rs485-wiring-guide
  → 关联产品页：/products/rs485-humidity-sensor-xyz200
  → 搜索意图：信息型
  → 月点击量：1,500
  → 优先级：P1
```

### 2.2 产品页现状审计

用 Screaming Frog 或 Ahrefs Site Audit 对所有产品页爬取，检查以下项目：

| 检查项 | 正常状态 | 常见问题 |
|---|---|---|
| Title 标签 | 含核心关键词，55~60 字符 | 空白、用产品名默认填充、过长 |
| Meta Description | 含卖点+关键词，120~155 字符 | 空白或插件默认生成 |
| H1 标签 | 唯一、含主关键词 | 与 Title 完全相同、缺失、多个 H1 |
| 内链指入数量 | 建议 ≥5 个来自相关页面 | 0~1 个，全站孤立 |
| 正文字数 | 建议 ≥500 词（中文约 300 字） | 仅参数表格，50~100 字 |
| 图片 Alt 属性 | 含产品名/型号关键词 | 全部为空或 `image001.jpg` |
| Schema 标记 | Product + Offer | 完全缺失 |

把每个产品页的审计结果打分，按问题数量排序，优先改造关联博客流量最大的那些产品页。

## 三、内链漏斗建设：保住博客流量并向产品页导流

这是整个方案中 **ROI 最高的单项动作**，操作成本低，见效快（通常 2~4 周内 Google 重新抓取后可观察到产品页权重变化）。

### 3.1 内链植入原则

**原则一：语境相关性优先**

内链必须植入在正文中语义最相关的段落，而不是硬塞在文章末尾。Google 会评估内链的上下文语境，语境越相关，权重传递效率越高。

错误做法：
```
...本文介绍了温湿度传感器的选型要点。如需了解我们的产品，请点击这里。
```

正确做法：
```
...在食品冷链场景中，建议优先选择 IP67 防护等级、支持 Modbus 协议的传感器，
例如 XYZ-200 工业温湿度传感器，量程覆盖 -40 至 120℃，可直接接入主流 PLC 系统。
```

**原则二：锚文本使用关键词，不用泛化词**

| 类型 | 示例 | 建议 |
|---|---|---|
| ✅ 精确关键词锚文本 | `RS485 工业温湿度传感器` | 最优，直接告诉 Google 目标页主题 |
| ✅ 部分关键词锚文本 | `工业级温湿度传感器方案` | 次优，有语义但更自然 |
| ❌ 泛化词锚文本 | `点击这里`、`了解更多`、`查看产品` | 无权重价值 |
| ❌ URL 锚文本 | `https://xxx.com/products/xyz-200` | 可读性差，不推荐 |

**原则三：每篇博客内链数量控制**

- 每篇博客文章植入 **1~2 个产品内链**即可
- 不要在同一篇文章中多次指向同一产品页（重复内链 Google 只计算一次）
- 保留文章内原有的内链（指向其他博客的），不要删除

**原则四：不动博客的 URL、Title、H1**

这是保住现有排名的红线。即使发现某些博客 Title 写得不好，**在产品页改造完成并稳定前，不要修改博客页面的 Title 和 H1**，避免触发 Google 重新评估排名。

### 3.2 内链植入操作流程

1. 打开第 2 步整理的"博客-产品页映射表"
2. 按优先级从高到低，逐篇处理
3. 在 WordPress 后台打开文章编辑器（Gutenberg 或 Elementor）
4. 全文阅读，找到最适合插入产品内链的段落
5. 选中核心关键词 → 插入内链 → 保存更新
6. 在 GSC 中提交该博客 URL 的索引请求（加速 Google 重新抓取）
7. 记录操作日志（文章 URL、插入位置、锚文本、操作日期）

**操作日志模板：**

```
日期：2026-04-03
博客 URL：/blog/how-to-choose-industrial-sensor
插入段落位置：第 3 段（IP67 防护等级说明处）
锚文本：RS485 工业温湿度传感器
目标产品页：/products/rs485-xyz200
GSC 提交：已提交
```

### 3.3 博客底部 CTA 模块（标准化组件）

除了正文内链，每篇博客底部需要一个**与文章内容强相关的产品推荐模块**，这个模块同时承担三个作用：

① 给产品页传递内链权重  
② 将处于决策阶段的读者引导到产品页  
③ 降低跳出率，增加页面停留深度

模块 HTML 结构建议（可做成 WordPress 短代码或 Elementor 全局区块）：

```html
<div class="product-cta-block">
  <div class="product-cta-image">
    <img src="/wp-content/uploads/xyz200-thumbnail.jpg" 
         alt="RS485 工业温湿度传感器 XYZ-200" 
         width="120" height="120">
  </div>
  <div class="product-cta-content">
    <span class="cta-label">本文推荐产品</span>
    <h4 class="cta-product-name">XYZ-200 工业温湿度传感器</h4>
    <p class="cta-product-desc">
      RS485 接口 · Modbus RTU 协议 · IP67 防护 · 量程 -40~120℃ · 精度 ±0.3℃
    </p>
    <div class="cta-actions">
      <a href="/products/rs485-xyz200" class="btn-primary">查看产品详情</a>
      <a href="/contact?product=xyz200" class="btn-secondary">获取报价</a>
    </div>
  </div>
</div>
```

样式要点：与正文区域有视觉区隔（背景色、边框），但不要做成弹窗或遮挡内容，避免影响用户体验和 Core Web Vitals 评分。

### 3.4 漏斗层级补充：评估层页面

如果博客是认知层，产品页是决策层，那么中间缺少"评估层"——用户在了解基本概念后，进入"比较选项、评估适配性"阶段，这个阶段的内容目前是空白的，需要补充以下页面类型：

**① 应用场景页（Solutions Page）**

- URL 格式：`/solutions/食品冷链温湿度监控`
- 覆盖搜索词：`食品冷链传感器`、`冷链仓库温湿度监控方案`
- 内容结构：行业痛点 → 解决方案说明 → 推荐产品列表 → 成功案例摘要 → CTA
- 与产品页的关系：页面内嵌 2~3 个产品卡片，每个卡片指向对应产品页

**② 选型指南页（Buying Guide）**

- URL 格式：`/guides/工业传感器选型指南`
- 覆盖搜索词：`工业传感器怎么选`、`RS485 传感器选型`
- 内容结构：选型维度说明（精度/防护等级/接口/量程）→ 各维度解析 → 推荐产品矩阵 → CTA
- 这类页面在 Google 中很容易排进前 3 页，且用户意图更偏商业

**③ 对比页（Comparison Page）**

- URL 格式：`/compare/产品A-vs-产品B`
- 覆盖搜索词：`XYZ-200 和 ABC-300 哪个好`、竞品对比词
- 内容结构：两款产品核心参数对比表 → 适用场景分析 → 选购建议 → CTA

这三类页面不需要一次全部完成，按搜索量优先级分批上线，每月上线 2~3 个即可。

## 四、产品页 TDK 改造规范

TDK 是技术成本最低、见效最快的改造项之一，但需要严格遵循规范，不能随意填写。

### 4.1 Title 标签（网页标题）

**作用：** 告诉 Google 这个页面主要讲什么，是 SEO 权重最高的 On-Page 信号之一，同时直接显示在搜索结果中，影响点击率（CTR）。

**字数限制：** 英文 55~60 字符，中文 28~32 汉字（超出部分在搜索结果中会显示为省略号）

**推荐结构：**

```
核心关键词 + 产品型号/规格参数 + 品牌词 | 站点名
```

**各类产品页 Title 示例：**

| 产品类型 | 推荐 Title 写法 |
|---|---|
| 标准型传感器 | `RS485 温湿度传感器 XYZ-200丨量程-40~120℃丨XYZ品牌` |
| 系列产品 | `工业温湿度传感器系列丨RS485/4-20mA可选丨XYZ品牌` |
| 定制/行业专用 | `食品冷链温湿度传感器丨IP67防护 Modbus协议丨XYZ品牌` |

**Title 写作注意事项：**

- 核心关键词放在最前面，不要把品牌名放在最前（除非品牌知名度极高）
- 不要堆叠多个关键词：`温湿度传感器 RS485 传感器 工业传感器 温湿度检测`——这是典型的过度优化，会被 Google 降权
- 参数信息要真实，不要为了关键词填写虚假规格
- 同站点内每个产品页的 Title 必须唯一，不允许重复

### 4.2 Meta Description（元描述）

**作用：** 不直接影响排名，但显示在搜索结果标题下方，是影响用户是否点击的核心文案，直接决定 CTR（点击率）。CTR 是 Google 的间接排名信号。

**字数限制：** 英文 120~155 字符，中文 70~80 汉字

**推荐结构：**

```
产品核心卖点（1句）+ 关键技术参数（2~3个）+ 适用场景（1句）+ 行动号召（1句）
```

**示例：**

```
RS485 工业温湿度传感器，精度 ±0.3℃，量程 -40 至 120℃，IP67 防护，支持 Modbus RTU/ASCII 协议。
适用于食品冷链、制药仓储、暖通控制。现货供应，7 天发货，免费提供选型技术支持。
```

**Meta Description 写作注意事项：**

- 必须包含 1~2 个核心关键词，但要自然融入，不要生硬堆砌
- 行动号召词要具体（"获取报价"、"下载规格书"），不要用"了解更多"
- 如果产品有价格优势、服务承诺（如 7 天发货、1 年质保），可以写进去，能提升点击欲望
- 每个产品页的 Meta Description 必须唯一，不允许复制粘贴

### 4.3 Keywords 标签

Google 早已宣布不将 Keywords 标签作为排名信号，但 Bing、Baidu 仍有部分参考权重，站内搜索也可能使用。

建议填写 3~5 个核心关键词，不超过 8 个：

```
工业温湿度传感器, RS485 传感器, Modbus 温湿度传感器, 冷链传感器, XYZ-200
```

不需要花过多精力在此标签上，填完即可。

### 4.4 Canonical 标签（防重复内容）

如果同一产品在 WooCommerce 中存在多个 URL 变体（例如不同参数的 SKU 页面），必须设置 Canonical 指向主产品页，避免重复内容分散权重：

```html
<link rel="canonical" href="https://yoursite.com/products/rs485-xyz200/" />
```

Rank Math 和 Yoast SEO 均可在产品页编辑界面直接设置 Canonical URL。

### 4.5 Open Graph 标签（社交分享优化）

虽然不直接影响 Google 排名，但产品页被分享到 LinkedIn、微信等平台时，OG 标签决定展示效果。ToB 产品的行业分享频率较高，值得配置：

```html
<meta property="og:title" content="RS485 工业温湿度传感器 XYZ-200 | XYZ品牌" />
<meta property="og:description" content="精度 ±0.3℃，量程 -40~120℃，IP67，Modbus 协议，适用于工业现场温湿度监控" />
<meta property="og:image" content="https://yoursite.com/wp-content/uploads/xyz200-og.jpg" />
<meta property="og:type" content="product" />
```

OG 图片建议尺寸：1200×630px，包含产品图 + 型号 + 核心卖点文字。

## 五、产品页内容结构深化

这是改造中工作量最大但效果也最持久的部分。原有的"名称 + 参数表 + 几张图"结构需要按照以下完整框架扩充。

### 5.1 推荐的产品页完整结构

```
┌─────────────────────────────────────────┐
│  1. Hero 区                              │
│     产品名（H1）+ 核心价值主张 + 主图      │
├─────────────────────────────────────────┤
│  2. 核心优势区                            │
│     3~5 个差异化 Feature（图标+标题+说明） │
├─────────────────────────────────────────┤
│  3. 产品规格参数表                        │
│     原有内容优化升级                       │
├─────────────────────────────────────────┤
│  4. 产品图集                              │
│     多角度图 + 细节图 + 应用场景图         │
├─────────────────────────────────────────┤
│  5. 应用场景区                            │
│     2~4 个典型行业场景                     │
├─────────────────────────────────────────┤
│  6. 技术文档下载区                        │
│     规格书 / 说明书 / CAD 图纸 / 认证文件  │
├─────────────────────────────────────────┤
│  7. 常见问题 FAQ                          │
│     5~8 个高频问题（覆盖长尾关键词）       │
├─────────────────────────────────────────┤
│  8. 相关产品推荐                          │
│     同系列产品 / 配套产品（内链）          │
├─────────────────────────────────────────┤
│  9. CTA 转化区                            │
│     询盘表单 / 联系按钮 / 在线客服入口     │
└─────────────────────────────────────────┘
```

### 5.2 Hero 区：第一屏决定跳出率

Hero 区是用户进入页面后最先看到的区域，必须在 3 秒内传达三个信息：

1. 这是什么产品（H1 标题）
2. 它能解决什么问题或有什么核心优势（副标题/标语）
3. 长什么样（产品主图）

**H1 写法规范：**

- H1 必须包含主关键词，但要自然，不能只是关键词堆叠
- H1 可以与 Title 标签不同（Title 是给搜索引擎看的，H1 是给人看的）
- 例：Title = `RS485 温湿度传感器 XYZ-200 | 量程-40~120℃ | XYZ品牌`；H1 = `XYZ-200 工业温湿度传感器，专为恶劣工业环境设计`

**副标题（核心价值主张）：**

用一句话说明产品的核心差异化点，格式建议：

### 5.3 核心优势区：传递差异化价值

这个区块用于向用户和 Google 解释"为什么选这款产品"，通常由 3~5 个 Feature 卡片组成，每个卡片包含：图标 + 短标题（H3）+ 2~3 句说明文字。

- 聚焦用户真正关心的维度，而非什么参数都列

### 5.7 FAQ 模块：最容易被忽视的长尾流量金矿

FAQ 是产品页中**最容易覆盖长尾关键词、且最容易抢占 Google "People Also Ask"（相关问题）展示位**的区块。

**FAQ 题目的来源：**

1. 在 GSC 中查看产品页有哪些已有展示但点击量低的查询词，这些词很可能是 FAQ 问题的原型
2. 在 Google 搜索核心词，查看搜索结果页底部的"相关搜索"和 PAA 框
3. 整理客户询盘中最常见的重复问题（技术支持人员是最好的 FAQ 题目来源）
4. 在 Answer The Public（answerthepublic.com）或 AlsoAsked 中查询该类产品的常见问题

**FAQ 问题写法规范：**

- 直接使用用户会搜索的完整问句（不要缩写或转述）
- 答案控制在 50~150 字，直接回答，不要绕弯子
- 每个 FAQ 答案中自然包含 1~2 个相关关键词

**示例 FAQ（8 条）：**

```markdown
**Q1：RS485 接口温湿度传感器和 4-20mA 型号有什么区别，怎么选？**

RS485 型适合多点联网场景，一条总线可挂载最多 32 台，适合仓库、厂房等需要集中
监控多个点位的场合；4-20mA 型为模拟量输出，抗干扰能力强，适合单点接入 PLC
模拟量模块或远距离传输（≤500 米）。如果已有 PLC 且带 RS485 通信模块，
优先选 RS485 版本，成本更低，扩展更方便。

**Q2：XYZ-200 传感器支持哪些通信协议，能接入西门子 PLC 吗？**

支持 Modbus RTU 和 Modbus ASCII 两种协议，波特率可在 1200~115200 bps 之间
通过拨码开关设置。可以直接接入西门子 S7-1200/1500（通过 CM 1241 通信模块）、
三菱 FX5U（内置 RS485 口）、施耐德 M340/M580 系列 PLC，我们提供这几款 PLC
的示例程序下载。

**Q3：IP67 是什么级别的防护，可以直接用水冲洗吗？**

IP67 是 IEC 60529 标准定义的防护等级：第一位数字 6 表示完全防尘，第二位数字
7 表示可短暂浸入 1 米深水中（30 分钟）。日常清洁可以用水冲洗，但不建议使用
高压水枪直接冲射，以免损坏接线端子。食品加工或制药洁净室的日常 CIP 清洗环境
下可正常使用。

**Q4：传感器精度 ±0.3℃ 是出厂值还是全量程值，多久需要重新标定？**

±0.3℃ 为全量程（-40~120℃）内的精度保证值，非出厂测试值。在正常工作环境下，
建议每 2 年进行一次计量校准；如用于食品 HACCP 或药品 GMP 等合规场景，
建议每年校准一次，我们提供有偿年度校准服务，并出具 CNAS 认可校准证书。

**Q5：一台计算机或 PLC 最多可以挂多少台传感器？**

RS485 总线理论上可挂载 32 个节点，通过增加 RS485 中继器可扩展至 256 个。
实际部署中建议每段总线不超过 24 台，以保证通信稳定性。如需超过 32 台，
我们也提供 4G/NB-IoT 无线版本，无需布线，可接入云平台集中管理。

**Q6：传感器安装方式有哪些，可以安装在管道或风管内吗？**

标准版提供壁挂安装（含安装孔）和导轨安装两种方式。如需安装在管道内或
风管内（HVAC 场景），可选配专用的管道插入式安装支架（型号 XYZ-MT01），
支架适配 DN15~DN100 管径，另有 75mm 风管探头加长版可选。

**Q7：如果超出量程或长时间过载，传感器会损坏吗？**

温度传感器的安全工作范围为 -50~130℃，超出此范围长时间工作可能导致传感芯片
永久漂移。湿度传感器建议避免长时间暴露在 ≥95%RH 的凝露环境中（短时冷凝可
恢复）。产品配有过压保护电路，供电电压 10~30VDC 均可正常工作，超过 35V 时
保护电路动作断开，不会损坏主芯片。

**Q8：如何获取 Modbus 寄存器地址表和 PLC 示例程序？**

所有技术文档（规格书、Modbus 寄存器表、西门子/三菱/施耐德示例程序）可在
本页面"技术文档下载"区块直接下载，无需注册。如需定制通信协议或有集成问题，
可通过页面下方联系表单提交技术支持请求，工程师 1 个工作日内响应。
```

### 5.8 相关产品推荐（内链网络建设）

此区块的核心价值不仅是提升用户体验，更是在产品页之间建立**双向内链网络**，使 Google 能够理解站点的产品体系结构，从而提升整体产品线的话题权威性（Topical Authority）。

推荐展示逻辑：

| 推荐类型 | 示例 | 内链价值 |
|---|---|---|
| 同系列不同规格 | XYZ-200 → XYZ-300（更高精度版）| 帮助用户在产品线内做决策 |
| 配套附件/配件 | 传感器 → RS485 转换器 | 提升客单价，建立产品关联 |

| 常见替代型号 | 当前型号 → 更适合某特定场景的型号 | 降低用户流失到竞品的概率 |

每个产品卡片包含：产品缩略图 + 型号 + 1 行核心差异说明 + 链接。展示 3~4 个即可，不要超过 6 个（太多会稀释注意力）。

### 6.3 WordPress 中如何添加 Schema

**方法一：使用 Rank Math（推荐）**

Rank Math 对 WooCommerce 和自定义 CPT 产品页有内置的 Product Schema 支持，在产品页的 Rank Math 设置面板中填写对应字段即可自动输出。FAQ Schema 也有专用的 Gutenberg 区块，直接添加"FAQ 区块"，Rank Math 会自动生成对应的 Schema。

**方法二：在 functions.php 中手动输出**

```php
function add_product_schema() {
    if ( is_singular( 'product' ) ) {
        $product = wc_get_product( get_the_ID() );
        $schema = array(
            '@context' => 'https://schema.org',
            '@type' => 'Product',
            'name' => get_the_title(),
            'description' => $product->get_description(),
            'sku' => $product->get_sku(),
            'offers' => array(
                '@type' => 'Offer',
                'price' => $product->get_price(),
                'priceCurrency' => get_woocommerce_currency(),
                'availability' => $product->is_in_stock() 
                    ? 'https://schema.org/InStock' 
                    : 'https://schema.org/OutOfStock',
            ),
        );
        echo '<script type="application/ld+json">' 
             . wp_json_encode( $schema ) 
             . '</script>';
    }
}
add_action( 'wp_head', 'add_product_schema' );
```

## 八、常见误区与风险规避

### 8.1 不要动博客的 URL 和 Title

即使发现某些博客的 URL slug 设计不合理，或 Title 关键词覆盖不佳，**在产品页改造未完成并稳定前不要修改**。修改博客 URL 必须做 301 重定向，且重定向后排名通常需要 4~8 周才能恢复，这段时间内流量会有明显下滑。

如果确实需要修改博客 URL，操作步骤：

1. 在 WordPress 后台安装 Redirection 插件
2. 新建重定向规则：旧 URL → 新 URL，类型选 301
3. 在 GSC 中提交新 URL 的索引请求
4. 检查站内所有指向旧 URL 的内链，批量替换为新 URL

### 8.2 不要批量复制粘贴 TDK

如果多个产品页的 Title 和 Meta Description 相同或高度相似，Google 会将它们视为重复内容，导致这些页面互相竞争排名（关键词蚕食，Keyword Cannibalization），最终都排不好。

**检查方法：** 用 Screaming Frog 爬取全站，在"Page Titles"和"Meta Description"标签页筛选重复内容，逐一修改。

### 8.3 不要过度优化锚文本

如果所有内链都使用完全相同的关键词作为锚文本（例如全站 50 个内链都是"RS485 温湿度传感器"），Google 会认为这是人工干预，存在过度优化风险，可能触发 Google Penguin 算法降权。

**健康的锚文本分布：**

- 精确关键词锚文本：20~30%
- 部分关键词/变体锚文本：40~50%
- 品牌词锚文本：10~15%
- 自然语言锚文本（"这款传感器"、"该产品"）：10~20%

### 8.4 不要在产品页堆砌关键词

产品页正文中自然出现关键词 3~5 次即可（包括 H1、H2、正文段落），不需要强行重复。Google 的语义理解能力已经非常强，过度重复不会增加权重，反而可能触发过度优化惩罚。

### 8.5 Schema 数据必须与页面内容一致

Schema 中标注的价格、库存状态、评分数量必须与页面实际显示内容一致。如果 Schema 标记"In Stock"但页面显示"暂时缺货"，Google 会在 GSC 中报告 Schema 错误，严重时会撤销富片段展示资格。

## 十、总结

ToB WordPress 电商站从"博客引流"走向"产品页自然排名"的核心路径是：

> **内链建设（权重传导）→ TDK 规范（搜索信号）→ 内容深化（停留时长+长尾覆盖）→ Schema 标记（富片段展示）→ 漏斗层级补全（评估层页面）**

这五个步骤不需要同时完成，按优先级分阶段推进，第一阶段仅需 2 周、零外包成本，就能让 Google 看到明确的产品页权重信号，后续数据会随着内容深化持续改善。

改造的全程原则是：**不打破现有博客排名，在存量权重基础上向产品页分配流量和权重**，而不是重建站点或大量制造新内容。这是 ToB 电商 SEO 性价比最高的增长路径。
