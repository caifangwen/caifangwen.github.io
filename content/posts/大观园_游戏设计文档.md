## 1. 游戏核心设定

### 1.1 世界观

《大观园》以《红楼梦》为背景，是一款**百人异步社交策略游戏**。玩家扮演贾府中的各色人物，在共同的家族命运容器里，以截然不同的视角和手段，争夺资源、名声、情报与最终的生存评分。

**这不是谁杀死最多人的竞技游戏，而是一个末世生存游戏。** 家族的倒塌是注定的，问题只在于——你是推倒它的人，还是最后一个逃离的人？

### 1.2 异步博弈的时间架构

| 时间单位 | 对应现实时间 | 发生的事 |
|---------|------------|---------|
| 一日 | 约 2 小时 | 精力恢复、月例发放、情报触发 |
| 一旬 | 约 20 小时 | 诗社结算、管家审批清单刷新 |
| 一月 | 约 3 天 | 账目平衡检查、突发事件触发概率上升 |
| 一局 | 约 10–15 天 | 家族兴衰完整周期，末日倒计时启动 |

玩家不需要同时在线——每次登录，系统将"这段时间世界发生了什么"以**送来的帖子、收到的眼色、流言碎片**等形式呈现。

### 1.3 两个全局数值：末日进度条

| 数值 | 增加原因 | 后果 |
|-----|---------|-----|
| **亏空值** | 管家贪污、预支月例、奢靡采办 | 达到上限触发"财政危机"，减少公中拨款 |
| **内耗值** | 玩家举报、公开争斗、丑闻发酵 | 达到上限触发"抄家清算"，提前结束本局 |

> **核心博弈：** 沉船派玩家希望加速内耗值，在抄家前变现离场。保家派玩家则需要花费资源降低内耗值。双方博弈贯穿全局。

## 2. 五大玩家阶层

| 阶层 | 人数 | 原型角色 | 核心玩法 |
|-----|-----|---------|---------|
| **管家** | 5人 | 王熙凤、探春、赖大 | 资源分配、账目操控 |
| **主子** | 20人 | 宝玉、黛玉、宝钗、三春 | 名望竞争、外交联盟 |
| **丫鬟小厮** | 60人 | 晴雯、袭人、茗烟 | 情报交易、草根逆袭 |
| **元老** | 3人 | 贾母、王夫人 | 最高裁决权，厌弃与赏赐 |
| **清客游民** | 12人 | 詹光、刘姥姥、马道婆 | 搅局、预言、诅咒 |

### 阶层流动规则

- 上一局积分最高的玩家优先获得**元老**阶层资格
- 丫鬟达到特定条件可晋升为**姨娘**（主子阶层）参与下一局
- 管家威望归零可被**强制降级**，由备选玩家接替

## 3. 管家系统 · 资源博弈

### 3.1 公中银库

每日系统根据家族繁荣度拨发"公中银子"。管家掌握全部分配权。

#### 分配操作表

| 操作 | 效果 | 风险 |
|-----|------|-----|
| 标准月例发放 | 按角色等级足额发放 | 无，但无个人收益 |
| **暗中克扣** | 截留差价转为私产 | 超过3人同时被克扣触发"告状风险" |
| **额外赏赐** | 目标好感度+15，发出"知心帖" | 亏空值+2 |
| **截留道具** | 采办物资20%留存为私产 | 若清账时发现，属于账目漏洞证据 |

#### 两本账机制（新）

管家可维护**明账**（对外公示）与**暗账**（真实数据）。清算时，若调查方获得"暗账碎片"，可发起"对比审查"，瞬间暴露全部漏洞。

### 3.2 批条系统 · 每日行动

管家每日拥有 **6点精力**，以下行动均消耗精力：

| 行动 | 精力消耗 | 效果 | 社交影响 |
|-----|---------|-----|---------|
| 采办物资 | 1 | 获取稀缺道具分配权 | 道具接收方获属性提升 |
| 差使分派 | 1 | 指定玩家执行差事 | 被派者精力+2消耗，获少量银两 |
| 搜检大观园 | 2 | 随机搜查5–10名玩家 | 可搜出情书、私约、账目碎片 |
| 预支批条 | 1 | 允许某玩家提前支取 | 好感度大幅提升，亏空值+5 |
| 平息流言 | 2 | 消除一条正在发酵的流言 | 被帮助者欠人情 |
| 封锁消息 | 3 | 阻止某条情报传播12小时 | 高代价，保护重要机密 |

### 3.3 查账机制

```
发起阶段
  └─ 任意玩家消耗 20点气数 + 至少1份账目漏洞碎片 → 发起查账请求

调查阶段（24小时异步）
  └─ 元老玩家异步审理
  └─ 管家可在此期间花费资源"销毁证据"或"拉拢证人"

裁决阶段
  ├─ 亏空 ≤ 10%   → 无罪释放，举报者损失20点气数
  ├─ 亏空 10–30%  → 管家降级，罚没当期所有私产
  └─ 亏空 > 30%   → 抄家事件提前触发
```

**反咬机制（新）：** 管家被查时，可消耗手中的"情报碎片"反咬举报者，将调查转移到举报玩家身上，拖延时间，制造混乱。

### 3.4 言官系统 · 民意制衡

每次管家做出重大分配决策后，全体玩家有 **6小时** 投票窗口：

| 不公票比例 | 后果 |
|----------|-----|
| ≥ 20% | 管家威望值-15，该批次决策可被撤销 |
| ≥ 40% | 元老有权启动"罢免审议" |
| 威望跌破临界线 | 批条指令失效，由备选管家代理3日 |

### 3.5 管家双路线终局

| 路线 | 条件 | 奖励 |
|-----|------|-----|
| 🌿 **贤能管家** | 家族亏空≤15%，平均满意度≥70%，无告状事件 | "治理名望"勋章 + 下一局管家优先资格 |
| 🐍 **末世枭雄** | 家族覆灭，个人私产≥500两，抄家前成功转移资产 | "乱世奸雄"勋章 + 大额现金奖励 |

## 4. 丫鬟小厮系统 · 情报经济

### 4.1 听壁脚 · 情报获取

丫鬟每日可选择**挂机监听一个场景**：

| 场景 | 可能获取的情报类型 | 价值 |
|-----|----------------|-----|
| 怡红院后窗 | 宝玉的私人行动、赠礼记录 | 高 |
| 管家后账房 | 账目漏洞碎片、批条草稿 | 极高 |
| 蜂腰桥 | 丫鬟间的交易记录、私约 | 中 |
| 荣国府大门 | 外来访客身份、礼单内容 | 中高 |
| 贾母处 | 元老喜好偏向、赏赐对象 | 极高（稀有触发）|

**双人挂机（新）：** 与另一名丫鬟结成**对食**或**私约**关系后，可同时监听两个场景，情报获取速率翻倍。但一旦有人背叛，对方可以用共同获取的情报同时出卖你。

### 4.2 狐假虎威 · 借势系统

| 主子权势 | 你的行动范围 | 风险 |
|---------|-----------|-----|
| 高（如凤姐当权） | 可传话、要账、代替主子出席场合 | 主子倒台时一起受牵连 |
| 中（普通主子） | 可在园内自由走动、传递口信 | 无特别风险 |
| 低（主子被厌弃） | 行动受限，搜检概率上升 | 考虑另觅主子 |

**弃主投靠（新）：** 丫鬟可主动脱离当前主子转投他人。旧主子记录"被背叛标记"。背叛次数过多触发**"声名狼藉"**状态，没有任何主子愿意接收。

### 4.3 传话系统 · 信息操控

底层玩家是唯一能跨阵营私聊的桥梁，每次充当信使有三个选择：

| 选择 | 效果 | 代价 |
|-----|------|-----|
| ✅ 原话带到 | 获得发件方小费，双方好感度+5 | 收益最低 |
| ⚠️ 篡改口信 | 制造误会，引发目标玩家错误决策 | 若被发现，永久失去双方信任 |
| ❌ 截留信件 | 日后可作为要挟或出卖的筹码 | 双方关系归零，触发"鸿雁断线"事件 |

### 4.4 流言工坊 · 信息战

消耗 **5点精力** 可发布一条流言：

```
0–6小时：口耳相传
  └─ 目标玩家消耗10点气数可直接压下，不留痕迹

6–12小时：人尽皆知
  └─ 需管家消耗2点精力才能平息
  └─ 目标玩家"体面值"-5

12小时后：板上钉钉
  └─ 变成"实质处罚"，名誉值-15，无法消除
  └─ 内耗值+3
```

**流言嫁接（新）：** 若手中有同一玩家的两条情报碎片，可将其合并为一条流言，杀伤力翻倍，且难以证伪。

### 4.5 三条逆袭路径

| 路径 | 条件 | 终点奖励 |
|-----|------|---------|
| 🌸 **首席大丫鬟** | 忠诚度≥90，处理突发事件≥5次 | 解锁"副主子权限"，可影响主子决策 |
| 💫 **收房/姨娘** | 触发主子专属剧情，累计互动≥20次 | 下一局以"主子"阶层开局 |
| 🚪 **赎身出府** | 个人积蓄≥300两，抄家前成功转移资产 | 本局最高评分之一，"全身而退"成就 |

## 5. 主子系统 · 诗社与名望

### 5.1 海棠诗社 · 名望护城河

诗社每旬举办一次，全体玩家均可参与评赏：

| 参与方式 | 条件 | 收益 |
|---------|------|-----|
| 提交作品 | 消耗"才情值" | 获赞越多，名望值越高 |
| 匿名点赞 | 消耗"好感度" | 与被赞者关系提升，可作为暗中拉拢信号 |
| 匿名差评 | 消耗"气数" | 降低目标名望，高风险（可能被追查） |

#### 名望的实际价值

- 名望 ≥ 80：管家搜检你时消耗双倍精力
- 名望 ≥ 120：可发起"联名谏言"，强制撤销管家某项决策
- 名望 ≥ 150：元老被动关注你，赏赐概率大幅提升
- 名望 ≤ 20：任何人可以随意诋毁你，不受惩罚

### 5.2 代笔黑市（新）

才情值低的玩家可以花钱雇丫鬟玩家代笔投稿。代笔方可以选择：
- **真心帮忙**：收钱，主顾名望提升
- **故意写烂诗**：让雇主出丑，赚取额外的"坐收渔利"分数

## 6. 元老与清客系统

### 6.1 元老系统 · 审判生态

- **定夺权：** 两个玩家产生不可调和纠纷时，案件进入元老异步审批后台
- **厌弃机制：** 可对任意玩家施加"厌弃"状态，被厌弃者所有公共场合收益减半
- **额外赏赐：** 决定谁能获得系统随机刷新的顶级道具

### 6.2 清客系统 · 搅局生态

| 技能 | 效果 | 消耗 |
|-----|------|-----|
| 🔮 **预视** | 查看本旬最可能触发的突发事件类型 | 完成依附任务后解锁 |
| 🪡 **扎小人** | 目标下次重大决策时，30%概率出现"意外失误选项" | 30点气数 |
| 🌸 **送符咒** | 目标下次遭举报时，举报成功率-20% | 20点气数+贿赂道具 |
| 📯 **走街串巷** | 在"蜂腰桥集市"散布情报，无法追溯来源 | 10点精力 |

**刘姥姥进大观园事件（新）：** 当清客玩家在园内蹭饭次数达到阈值，触发全局事件——所有主子必须接待，消耗宴席资源。若礼遇得当，获得"民间口碑"加成，降低家族内耗值。

## 7. 突发事件设计

突发事件是打破固定节奏的关键机制，每旬随机触发 1–2 个，要求**全体玩家在48小时内做出异步响应**。

### 稀有事件

#### 🏮 元妃省亲
- 管家需在72小时内完成"接驾预算"，亏空值+25
- 主子阶层需准备才艺（诗词/题额），名望争夺激烈
- 丫鬟小厮差使全开，精力消耗翻倍但薪酬也翻倍
- **隐藏博弈：** 管家可在接驾预算里侵吞5–15%作为"跑腿费"，因为此时没人有空查账

#### ⚰️ 大出殡（秦可卿/贾敬薨逝）
- 家族进入丧期，禁止娱乐活动（诗社暂停1旬）
- 月例暂停发放
- 开启"奔丧来客"机制：外来玩家（清客）可携带外部资源进入家族循环

#### 🔥 抄检大观园
- 当内耗值达到80%时随机触发
- 对全部玩家进行强制搜检，所有背包内容全部曝光
- 本局的"大洗牌时刻"，是最高风险也是最高收益的时间节点

### 中等事件

#### 🔍 贾政大检查
- 所有玩家背包中的违禁品须在24小时内转移或销毁
- 管家若收到贿赂可以"放水"跳过某些房间的检查

#### 👵 刘姥姥进大观园
- 所有主子必须腾出资源接待
- 接待分足够高：获得"民间口碑"奖励，内耗值-8
- 背地里嘲笑刘姥姥的玩家若被情报碎片记录，名望-10

## 8. 清算逻辑与胜利条件

### 8.1 最终评分公式

```
最终得分 = (命运逆转分 × 命运系数) + 社交资本 + 个人财产 - 业障清算 + 结局彩蛋加成
```

### 8.2 五个评分维度

| 维度 | 计分逻辑 | 主要受益阶层 |
|-----|---------|-----------|
| **① 命运逆转分** | 与角色命运原著反差越大，分越高。抽到晴雯但活到最后且身价颇丰 → 满分 | 所有人 |
| **② 社交资本** | 统计有多少玩家在自己"亲信/爱慕"名单里填写了你（每人最多标注5人） | 主子/管家 |
| **③ 个人财产** | 直接计算账面和道具总值。抄家时成功转移的资产折算双倍 | 管家/丫鬟 |
| **④ 业障清算** | 每一次背叛盟友、陷害成功的记录均扣分。若被害方从未发现，业障值为零 | 所有人 |
| **⑤ 结局彩蛋** | 完成特定隐藏成就，如独自达成"赎身出府"或"首席大丫鬟"路线 | 丫鬟/清客 |

## 9. 技术选型：Godot 4

### 9.1 为什么选 Godot 4

| 理由 | 说明 |
|-----|------|
| **完全免费** | MIT协议，无版税，无引擎分成 |
| **GDScript 易学** | 语法接近Python，上手门槛低 |
| **Web导出** | 可直接导出为HTML5，玩家无需下载安装 |
| **场景系统** | 每个游戏界面（银库、批条墙、流言广场）都可以做成独立场景，模块化开发 |
| **信号系统** | Godot的信号机制天然适合异步事件驱动（"流言发酵完成"→触发惩罚） |

### 9.2 推荐技术架构

```
Godot 4 (前端 + 游戏逻辑)
    │
    ├── 场景树
    │   ├── Main.tscn          # 主入口，处理登录状态
    │   ├── Hub.tscn           # 大观园主界面（导航中心）
    │   ├── Treasury.tscn      # 银库界面（管家专用）
    │   ├── Inbox.tscn         # 门房收件箱（批条/帖子）
    │   ├── IntelBag.tscn      # 情报背包（丫鬟专用）
    │   ├── RumorBoard.tscn    # 流言广场
    │   ├── PoetryHall.tscn    # 诗社（主子专用）
    │   └── Settlement.tscn    # 清算结算界面
    │
    └── 后端 (Firebase Realtime Database)
        ├── /games/{game_id}   # 全局游戏状态
        ├── /players/{uid}     # 玩家数据
        ├── /treasury          # 公中账本（明账）
        ├── /actions_queue     # 待处理批条队列
        ├── /rumors            # 流言及发酵状态
        └── /events            # 突发事件日志
```

### 9.3 关键技术点

**异步通知（Godot + Firebase）**

```gdscript
# 监听流言发酵状态变化
func _ready():
    Firebase.Database.get_node("rumors").child_changed.connect(_on_rumor_updated)

func _on_rumor_updated(snapshot):
    var rumor = snapshot.get_value()
    if rumor["ferment_stage"] == 3:
        # 触发实质处罚，扣除目标玩家名誉值
        apply_reputation_penalty(rumor["target_uid"], 15)
```

**精力系统（本地计算 + 服务器验证）**

```gdscript
# PlayerState.gd - 单例
var stamina: int = 6
var last_refresh_time: int

func get_current_stamina() -> int:
    var elapsed = Time.get_unix_time_from_system() - last_refresh_time
    var recovered = int(elapsed / 7200)  # 每2小时恢复1点
    return min(stamina + recovered, 6)
```

// Firebase Cloud Function
exports.fermentRumors = functions.pubsub
  .schedule('every 30 minutes').onRun(async () => {
    const rumors = await db.ref('rumors').once('value');
    rumors.forEach(rumor => {
      const age = Date.now() - rumor.val().created_at;
      if (age > 43200000) { // 12小时
        applyRumorPenalty(rumor.val());
      } else if (age > 21600000) { // 6小时
        updateFermentStage(rumor.key, 2);
      }
    });
});
